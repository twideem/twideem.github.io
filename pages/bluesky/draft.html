備記

Apache OpenOffice

2011

Sun Microsystems 被收購後，OpenOffice.org 社群的一部分開發人員，在發展上受到 Oracle 商業思考的限制，於是出走成立 The Document Foundation，自行開發 LibreOffice，於是 OpenOffice 產生兩種分支。

由於 The Document Foundation 更受到 GPL 教徒們的信賴，眾多 Linux 發行版也指定選用 LibreOffice，聲勢浩大，沒幾年就邊緣化 Oracle 所屬的 OpenOffice.org 社群。沒面子的 Oracle 最後把 OpenOffice 捐贈給 Apache 組織，成為 Apache OpenOffice。

一冠上 Apache，那志工的熱忱與奉獻等形象整個浮現上來，馬上就擺脫臭名，於是現在我們不知道該選擇 Apache OpenOffice 好？還是 LibreOffice 好？

以目前來講，LibreOffice 3.6.5 在軟體執行效率上，以及功能操作上，都顯得更加用心在作設計，也就是在軟體本身求進步。而 OpenOffice 3.4.1 則是在應用細節上作調整，也就是軟體本身沒什麼進展，但在既有的操作方式上，能滿足更多使用者想要做的事。

我認為 Apcche OpenOffice 做法是更可取的！因為讓軟體本身有更多樣化的設計，其實不見得是使用者需要的，反而在使用者已經受用的設計上，加以修正、調整，令其更符合使用上的需求，會是更有工作效益的事。

但現階段來講，OpenOffice 3.4.1 修正的細節不夠多，但 LibreOffice 3.6.5 追加的功能卻很豐富，所以我還是推薦 LibreOffice……絕對不是因為我本身支持 GPL 的關係，絕對不是（心虛）。

2016

OpenOffice.org 捐給 Apache 後，並未吸引更多志工，幾年來更新速度慢到等於停滯，甚至有漏洞也沒足夠人力立即修補，一個月後才修補好漏洞。

態勢很明顯，OpenOffice 應該結束專案，讓大家用更好的 LibreOffice！

iThome online《知名開發人員：是不是該終結 OpenOffice 了》：
GNOME 開發者暨紅帽軟體工程經理 Christian Schaller 指出，讓使用者以為 OpenOffice 還活著且還在進化，只會傷害使用者對開放源碼應用軟體的評價。協助 OpenOffice 現有的用戶轉至 LibreOffice 才是有利大眾的事。」
OpenOffice 是過去很有名氣的辦公室軟體，它在 2000 年成為開放原始碼專案時，正好 Microsoft 大執法，幾乎都用盜版 Microsfot Office 的學生，像得到救贖一樣，有一套免費合法使用的辦公室軟體做為替代方案。往後十年，更推薦大家使用 *.odt 保存文書檔案，以免沒買 Microsoft Word 的人開不了 *.doc 文件。

沒在關注開放原始碼業界新聞的人，會以為 OpenOffice 比 LibreOffice 有名，然後選擇了品質較差的 OpenOffice，但其實 LibreOffice 才是有在進步的辦公室軟體。

名聲變臭的原因

OpenOffice 名聲變臭，是因為 Oracle 收購 Sun Microsystems 後，用自己的員工想將 OpenOffice.org 商業化，並刻意冷落原公司中負責 OpenOffice.org 社群的員工，好像不下指示就能避免他們擴張版圖一樣。

2010 年，遲遲等不到指示的 OpenOffice.org 社群，另外成立 The Document Foundation，好繼續制定目標努力推廣 OpenOffice.org。結果 Oracle 禁止 The Document Foundation 使用 OpenOffice.org 的商標。注意！不是 StarOffice 的商標，而是開放原始碼後，社群所制定的 OpenOffice.org 名稱與標示，被 Oracle 限制商標用途。並要求參與 The Document Foundation 的成員退出 OpenOffice.org 社群，這種基於公司利益打壓開放原始碼社群的行徑，誰看了都很氣憤。

可笑的是，大咖們通通照 Oracle 要求退出 OpenOffice.org 社群，加入 The Document Foundation 一起為 LibreOffice 寫程式更新功能，反觀 OpenOffice.org 是停擺的。明明 The Document Foundation 歡迎 OpenOffice.org 分享他們對 LibreOffice 開放原始碼的修改結果，OpenOffice.org 也不願接受分享，自己封閉起來慢慢發展，顯然完全受 Oracle 控制，已經稱不上是開放原始碼社群了。看到這種情況，所有 Linux 發行版都預裝 LibreOffice，而不是 OpenOffice.org，逐漸被邊緣化。

re:名聲變臭的原因

2011 年，拉不下臉的 Oracle 知道搞砸以後，並不是合併專案或停止專案，而是把 OpenOffice.org 捐給 Apache，擺明就是要讓 The Document Foundation 難堪。

看到這裡，Oracle 是一家怎樣唯利是圖的公司，應該不用多費唇舌。只是，這家公司為了自身利益，阻礙開放原始碼的進展，卻是不能不關心的話題。像是控告 Google 的 Android 侵犯 Java API 著作權，Oracle 想維護的真的是 Java 相容性？還是想分杯羹對每台 Android 裝置收取專利費？

Oracle 時代的 MySQL 和 Java，和 Sun Microsystems 時代，進步幅度一樣嗎？不一樣，都處在「別發展得太強勢，以免新世代來得太快。」而被壓制：「我這世代資料庫本業的利益還沒賺夠啊！不要亂來！」

PDFCreator

OpenOffice 3 開始，已經能精美地將文件匯出為 PDF 格式，與其使用 PDFCreator 間接產生 PDF 文件，不如直接以 LibreOffice 做為 PDF 編輯器，因此把 PDFCreator 列為評估選項。你慣用的排版軟體不支援 PDF 匯出功能再用 PDFCreator。

Evernote

隨手就能作筆記的軟體，此外還可分類、搜尋、標記，而且還跨機台，在個人電腦、行動電話、平板裝置都有 Evernote 可用，方便性與實用性都滿分……那為何放在「評估」裡面呢？

我認為 Evernote 對不適合打字的行動電話和平板裝置非常方便，可說是必須的！但我這份是個人電腦的免費軟體清單，個人電腦有大型鍵盤的關係，打字非常方便，所以選擇雲端服務來隨記反而比較乾淨俐落。

這也道出 App 和 Software 時代交替時，最明顯的分界吧？可攜性的電腦與辦公用的電腦，需要的程式會越來越不一樣。個人化的功能將以 App 來開發而越來越有趣，Software 將式微僅用來發展工作室用的工具。

新酷音輸入法

PIME 輸入法雖然更先進也更好用，但使用 Python 和 Node.js 開發的關係，每次開機要啟用輸入法時，作業系統會停頓好幾秒，所以 新酷音輸入法 還是有輕巧快速的優點。

新酷音輸入法的「使用 Shirt 快速切換中英文」「使用 CapsLock 切換中英文」只在編輯區字串下有效，只切換到中文狀態，卻未輸入中文字或全型符號，是按不出來的。

PeaZip 或 7-Zip

PeaZip 支援自己的 PEA 格式，它訴求封包安全，壓縮率小優於極致壓縮的 ZIP。但普及度不高，應該很少人會了網路傳檔安全而採用它。

此外支援 PAQ8 這高壓縮率的格式，但速度超慢，應該也很少人會為了讓檔案變小，卻得花一百倍的時間去壓縮和解壓縮，所以不實用。寧願花 10 分鐘下載 200MB 的壓縮檔，然後只要 2 分鐘就能解壓完畢，也不想花 5 分鐘下載 100MB 的壓縮檔，卻得 30 分鐘才能解壓。

壓縮率高、封裝速度又快、還可以視檔案種類切換演算法來最佳化的 7z，依然是最實用的壓縮格式！

通常來講，壓縮率排行為 PAQ > 7z > BZip2 > XZ > GZip > ZIP，其中：

．PAQ8 壓縮率比 7z 高 25% 以上，非常恐怖！
．7z、BZip2、XZ、GZip，前者依序比後者高 1%～2% 左右，相差不大。
．ZIP 差 GZip 10%，應該淘汰了～

雖然有 PeaZip 了，但還是值得推薦 7-Zip，因為以軟體來講它非常輕巧，而 PeaZip 顯得笨重。即使一次處理上百個檔案，7-Zip 依然爽快無比，但 PeaZip 一次處理數十個檔案就變得拖泥帶水了～

至於操作介面與檔案圖示太醜的問題，可以使用 7zTM 來解決。建議 7-Zip 9.25 + 7zTM 2.1（主程式）+ 7zTM 2.1.1（修正檔）的組合，不是版本越新就越好。

虛擬光碟機

截至 2014 年為止，虛擬光碟軟體，只有 DVDFab Virtual Drive 是沒有流氓行為的，其它都已經淪為惡意軟體了！缺點是不能模擬含 CD 音軌的遊戲光碟。但如果需求是掛載各家主流映像檔光碟的話，其實夠用了～

要掛載含音軌的遊戲光碟，只好用 DAEMON Tools Lite。但這已經變成「以置入軟體與廣告為條件，交換免費使用」的軟體，而且就大辣辣寫在安裝前的授權協議裡，所以往後新推出的版本，常常沒得選，要嘛被綁架然後安裝，不想被綁架就不給安裝。不然就是多方綁架，選不要修改瀏覽器首頁了，後面卻還有接二連三的項目要你選要不要，很容易就因為漏選而中標。而且每版的做法變來變去，安裝或升級新版時，務必非常謹慎。

要命的是，一但接受，就再也不能解除，例如植入惡意系統程序，只要偵測到開啟瀏覽器，就重新導引首頁，所以「改回首頁」沒用、在「新增移除程式」也找不到、「登錄檔」也摸不著頭緒，非常惡劣！

Krita

專業一點的 CG 畫家，想要免費繪圖軟體的話，首選應該是 Krita，但比較適合專業人士，外行人不太會使用。

所以本清單刊登易學易用的 Artweaver，每個人都能輕鬆用滑鼠畫出有筆觸的線條，描繪出不錯的 CG。

等上手後，正式將 CG 做為專業技能後，就改用 Krita 吧！

JPEGView

支援 Unicode 的看圖軟體中，我認為 JPEGView 是最棒的！放大圖片自動柔化所以不會有鋸齒，但切換圖片的效率比經常頓格的 Imagine 還要快。缺點操作介面過於乾爽（雖然換個角度其實這是它最棒的優點），連三秒鐘摸索的耐性都沒有的人，會覺得不容易上手，加上自訂化設定是用組態檔參數的方式操作（你可以親自輸入參數或者透過存檔的方式保存設定值），沒有直覺的圖形化介面，所以還是刊登 Imagine。

但純粹只想找個乾淨清爽看圖軟體來取代 Windows 內建程式的人，沒試過 JPEGView 你會後悔。

XnViewMP 則是長年處於 BETA 版，三不五時程式就當掉，所以不推薦。

XMedia Recode、HandBrake

XMedia Recode 比「格式工厂」支援更多新的格式，例如 WebM。而且轉出來的品質較高，可以調校的項目也較多，適合專業的使用者。缺點是不支援 Unicode，而且很容易當機，所以忍住不推薦。

HandBrake 是轉換影音檔案到不同平台裝置播放的軟體，換個角度來想，也能當影音轉換工具使用。不用複雜調校，就能轉出品質比 XMedia Recode 更高、檔案更小的影音檔案，更不會狂當機～

需要把 MIDI 轉為其他格式的話，可用「MIDI 轉換精靈」。

六角大王

六角大王免費版的官網載點：https://www.shusaku.co.jp/www/product_free.html，點進去發現官方打算結束這項產品的經營了！

而商業版的「六角大王 Super」則交由「株式会社セルシス」負責販售的業務。

六角大王是 3D 繪圖外行人也能順利製作 3D 模型的軟體，商業版則著重於更多懶人法的設計，像是載入自己的大頭照就能產生自己的 3D 繪圖人物，或者 2D 圖片轉成 3D 模型，都讓人津津樂道～

「六角大王 Super」的另一個賣點，就是售價非常便宜！以日圓換算成新台幣大約 2000 元（不過當年台灣代理時賣貴約一倍），以日本人的薪資水準，起薪換算成新台幣大約六萬元來說，根本便宜到爆！說「六角大王 Super」是軟體界最物美價廉的產品一點也不為過。

但這麼棒的軟體，依然走向經營不下去的命運，軟體是否適合「薄利多銷」的觀念，看來相當值得省思～

Movie Marker 或 Avidemux

Windows XP 開始，內建了很方便使用的影片剪輯軟體：Movie Maker。不過有點像 Microsoft 為推廣自家 WMV 格式的工具，那時我認為支援更多格式的 Avidemux 是更好的選擇，雖然剪輯功能較差，但修片功能較強。

到了 Windows 7，改由下載 Windows Live Essentials 2009 來安裝 Movie Marker。只為了安裝一款小小的工具，卻得下載整包上百 MB 的檔案不說，非常容易當機，而且新的 Ribbon 操作介面相當難用，並不吸引人專程下載安裝使用；甚至有人繼續拿 Windows Vista 的 Movie Maker 2.6 來用。

到了 Windows 8，改為看起來與先前沒兩樣的 Windows Essentials 2012 來安裝 Movie Marker，而且確實只是以先前設計為基礎，將缺點加以改善而已。不過，支援 MP4 格式，且可以根據各式裝置如 Android 平板或各家影片網站如 YouTube 來最佳化影片，不滿意的話，乾脆自訂你想要畫面尺寸、速率、框速，來壓出你覺得畫質既理想、檔案又很小的影片。更別說剪輯與修片等功能已經比 Avidemux 強大，而且操作簡單順手，至少剪輯得精準，不會發生明明切割至某個片段，出來的影片卻從其它片段開始。

因此現在我推薦 Windows Essentials 2012 Movie Marker 更勝於 Avidemux！

但 Avidemux 可以照原影片的格式，直接 COPY 你想要的片段，速度非常快！又完全不會破壞原影片的畫面表現！如果對「修片」沒興趣，純粹只想擷取原影片的其中一小段來保存，Avidemux 會是更好的選擇。前提是你不在乎剪得不夠精準的話。

Notepad++ 以外的純文字編輯器

Notepad++ 的特點是輕巧：檔案小，啟動俐落，而且具備強大的文字編輯能力，適合取代 Windows 內建「記事本」。但早期比較像是具備高亮度程式辨識功能的文字編輯器而已，第五版才開始持續強化有助於專案開發的功能，目前為止對「資訊工程師」來說仍會嫌它不足以打造為「整合開發環境」。這時連改都不用改就已經媲美 IDE 的 RJ TextEd 會是更好的選擇，能夠提升數倍的程式編寫時效。

PSPad 適合講究方便順手即可的「一般程式工作者」，它跟 Notepad++ 一樣檔案小，但整合出一個簡單又實用的開發環境，並且支援不少特定工作的輔助工具；雖然那些工具擬定的做法，很多已經過時了！但不妨試看看，說不定會是很好的選擇。

用起來跟 Notepad++ 一樣精巧又乾淨俐落的 MadEdit 則是具備跨 Unix like 和 Windows 平台的能力，但這部分恐怕用 Java 打造的 jEdit 會是更好的選擇，尤其它是所有純文字編輯器中，處理效率最高的，往往可以比別人快十倍的時間處理完同樣事情，缺點是只有英文。

最後，自訂能力最高的 GNU Emacs 是專業人士用的系統環境：「強大的程度已經超越 IDE，然後已經不能算是文字編輯器了！」說到 Emacs 這少數人才懂得好用的程式編輯器，習慣 vi 那字裡行間可用瞬間移動方式來編輯的人也是少數，他們頂多選擇 MS-DOS and MS-Windows 版本的 vim 來用，其他文字編輯器一概看不上眼。

Chrome、IE、Opera、Safari

首先，Chrome 許多機制，受到 Google 自己意識形態的影響，而採取綁死的設計，不讓使用者選擇要與不要。雖然是很棒的網頁瀏覽器，但隨著版本的推進，也越綁死了許多不是使用者真正想要的機制。

接著，雖然我很不想稱讚 IE，但 Windows 8 的 IE 10 確實超越其它網頁瀏覽器，成為處理網頁時速率最快的一款！主要原因在於 IE 10 並不只是著重 JavaScript 的飆速、或者軟體本身的性能，而是在排版引擎這部分也下足功夫。IE 10 從「整體上」考量一張頁面從載入到顯示的諸多環節，盡量搶在第一時間就將「其實還沒載入完畢的網頁」處理好交到瀏覽者手上，然後趁瀏覽者移動滑鼠要操作頁面上的功能這兩三秒時間，再來處理剩下來的工作，於是感覺上比其它網頁瀏覽器更早兩三秒就處理好一張網頁。這種像串流技術無縫銜接的體驗，的確是很棒的設計，這點是 Firefox 與 Chrome 都應該正視與投入的議題。

Opera 擁有最棒的人性化操作介面，而且甚至比 Firefox 更能自訂化適合自己的網頁瀏覽環境。但自行研發的排版引擎總是冷門，遇到未遵循 W3C 規範的網站，顯示出來網頁容易有問題。這種網站通常成立於早期、或受制於早期成立時留下的包袱。當時網頁設計的習慣，是用 IE 與 Netscape 進行測試兩邊是否正常即可，還沒有是否遵循 W3C 所制定 HTML 規範的思維；或者說即使遵循規範也沒用，因為市占率最高的 IE 一直都扮演破壞規範的角色，要在 IE 執行就常常要破壞規範。而以前 Opera 是屬於商用軟體，很少人會買來用，當時沒有人會吃飽沒事測試說網頁是否能正常顯示於 Opera。於是早期成立的網站，有些會發生既不相容 Opera、也不相容 W3C 制定 HTML 規範，導致 Opera 無法正常顯示的問題。不過 iThome 網站 2013 年 2 月 19 日的新聞報導：「Opera 預計今年針對個人電腦及智慧型手機所推出的瀏覽器版本多數都將採用 WebKit 核心，並表示，與其持續開發自己的瀏覽器核心，和開放源碼社群一同改善 WebKit 及 Chromium 顯得更有道理。」未來我可能會改推薦 Opera[1]。

最後，我這反蘋果的人，是不可能推薦 Safari 的。

[1] 結果 Blink 排版引擎的 Opera 15 推出後整個悲劇，簡直是自廢武功的版本，功能全部砍掉，幾乎完全無法客製化使用者想要的操作環境；原本功能最豪華 Opera 12 瀏覽器，一舉變成最陽春的瀏覽器。

KompoZer

舊文

很久以前，Netscape Communicator 內建一款網頁編輯軟體：Netscape Composer，開放原始碼專案的 Mozilla Suite 自然引用這款軟體。其後 Linspire 資助 Daniel Glazman 增益這款軟體的功能，發表為 Nvu 免費釋出，好讓 Linux 作業系統能有一款足以媲美 FrontPage 和 Dreamweaver 的網頁編輯軟體。

但 Mozilla Suite 不再發布新版本後，Nvu 也跟著宣布停止開發。Fabien Cazenave 修補 Nvu 剩餘的 Bug，發布為 KompoZer，並陸續強化這款軟體讓它與 Microsoft Office 的 FrontPage 不相上下。完成後就很少再更新。

2010 年，Daniel Glazman 又重新開發網頁編輯軟體，取名為 BlueGriffon，支援 HTML5 網頁的標準。

BlueGriffon 比 KompoZer 有更嚴謹的機制，讓設計出來的網頁符合 HTML5 的規範，這甚至是讓它足以堪稱第一網頁編輯軟體的特點！但以網站管理的功能性來講，不如 KompoZer 強悍。

由於 HTML5 依然不是使用時機，甚至它訴求的語意往往對現行技術習慣上造成問題；目前 HTML 4.01 穩定的時代，其實還是值得推薦更接近 FrontPage 的 KompoZer。

新增

HTML5 已經成熟，並廣泛被使用，因此無法正常顯示 HTML5 的 KompoZer 已經沒有存在價值。

希望繼續用 KompoZer 的人，可以選擇 SeaMonkey 的 Composer。

SeaMonkey 是接手以前 Mozilla Suite 1.7 繼續開發新版本的專案軟體，因此內含 Composer，而且支援編輯 HTML5 網頁的能力！

但「所見即所得」的部分，沒多少進步的 SeaMonkey Composer 已經明顯不如 BlueGriffon 來得順手，更別說 BlueGriffon 的「原始碼模式」還支援自動完成語法的功能，以及高亮度辨識。

因此本軟體清單選擇刊登 BlueGriffon，而不是由 SeaMonkey Composer 取代原先刊登的 KompoZer。

Free Download Manager

我已經有好幾次經驗，如果安裝 FDM 後保持預設值正常使用就沒事，但手動設定為自己覺得更理想的環境，然後下載一陣子，會忽然發生下載速度變慢的情況，降到每秒只剩 15KB～30KB，之後即使刪除軟體，或者怎麼嘗試設定乙太網路與建立的連線，電腦都將永遠以這龜速下載，最後只能靠系統還原來修復。

不過找不到證據是 FDM 把 Windows 的網路系統搞爛掉，讓我覺得這樣的說法很不負責任。

我所說變龜速的情況，只發生在下載檔案時，在「HiNet 連線速率測試」是正常的，甚至 HiNet 提供測試用的 HTTP 載點下載檔案時，也能飆到每秒 200KB（雖然我的網路應該每秒 450KB 左右才正常），但其他網站下載檔案都是 15KB～30KB 之間。

這樣的事好幾年來，已經在聯強 MiTAC 桌上型電腦（Windows XP）、Acer Aspire 4736ZG（Windows 7）、Lenovo G580（Windows 8）發生，都是在安裝 FDM 後把 Windows 的網路設定給搞壞掉。（這些年用的同樣都是 HiNet ADSL）

P2P 檔案共享軟體

這類軟體沒顧及到保護使用者的隱私，以 eMule 為例，該軟體在下載的同時也會上傳檔案，還將 IP 公開出來。

這樣的設計，在下載行為觸法時，就等於非法分享檔案，而且方便警方網巡根據 IP 找罪犯，在台灣已發生多起警方持搜索票上門逮人並查扣電腦的案件，上網 Google 就知道嚴重性，更別說歐洲國家故意藉由這樣的軟體網釣使用者犯罪來抓盜版，因此不建議再使用 P2P 檔案共享軟體！

私下檔案共享通常不會使用 P2P，而是用有金鑰保護的網路硬碟，因此使用 P2P 檔案共享軟體，就是為了下載盜版漫畫、影音、遊戲。

本站決定移除相關連結，以免誤導使用者觸法被抓。

Avira AntiVir Personal

俗稱小紅傘的 Avira AntiVir Personal，與 avast! Free Antivirus 的選擇，其實就像「卡巴斯基」與「賽門鐵克」一樣：掃毒能力強的 Avira 與系統防護完善的 avast! 該作何選擇？

在這網路攻擊比電腦病毒問題嚴重的時代，我選擇 avast!。

而且以免費版本來說，avast! 開放的功能較為慷慨，小紅傘的先進技術往往只開放給付費者使用。從 Avira 連超鏈結都沒打上去來看，其實我根本就不推薦評測數據漂亮所以口碑第一的小紅傘，或許它只是「實而不華」，但網路攻擊防不勝防的現況下，坦白說「華而不實」的軟體反而更有保障。

然而這顛覆傳統的價值觀念，使得小紅傘才是免費防毒軟體的第一位。

Defraggler 不建議使用

Defraggler 有非常快速的重組功能，這是我刊登這款軟體的原因。但我希望你只使用這個功能就好，不建議你拿來完整重組硬碟裡面的資料。

硬碟動輒幾百 GB 的時代，完整重組要花四個小時以上，然後你硬碟的磁頭就連續好幾個小時都在那邊快速讀寫檔案，是無謂的消耗。

真要完整重組的話，如果是裝載作業系統的硬碟，建議只重組 Windows 資料夾，然後其它的先搬到其它硬碟槽再搬回來。其它的硬碟槽也想完整重組的話，一樣把檔案先搬到其它硬碟槽再搬回來。這一樣可以讓檔案不再破碎，而且至少不用花四小時以上的時間。

總之，請避免讓電腦完整重組那麼久的時間！

Slax 或 Puppy Linux

兩者都是檔案小到可以放在 256MB 隨身碟，然後用 USB 開機就能進桌面，看要上網或打字都行的 Linux 作業系統！

Puppy Linux 檔案小將近一半，但只有英文版，不過進入作業系統後，應該可以從官方下載套件進行中文化，沒有的話，可以自行到 amtbrobert 的協作平台下載繁體中文化 pet 外掛套件。然後，用 Universal USB Installer 將 ISO 檔製作成開機隨身碟即可。

Slax 直接提供 USB 版可以下載，重點是有中文版，而且提供的桌面環境比較美觀。使用 USB 版的話，建議將隨身碟格式化為 FAT32 再放檔案，避免開機時無法辨識。然後，執行 boot 資料夾裡面的 bootinst.bat 即可成為 USB 開機碟～

Universal USB Installer 或 UNetbootin

兩者都是將安裝光碟映像檔轉為 USB 開機隨身碟的工具。

轉 Linux Live 光碟的 ISO 時，選 Universal USB Installer 可以同時格式化 FAT32，轉換過程一步到位，完成之後幾乎都能開機不會失敗！

選 UNetbootin 的話，事先有些程序漏掉沒做好，轉換出來的隨身碟開機之後會找不到安裝系統，又要再花十幾分鐘時間轉換。但 UNetbootin 設計了自己的 USB 開機選單，一些非 Linux、甚至非作業系統的 ISO 開機光碟，可以嘗試用它來製作成 USB 開機隨身碟。

BlueStacks Gaming Platform、NOX App Player、KOPLAYER

NOX App Player 訴求是為所有電腦平台最佳化，因此最穩定：相容性最好，執行效率也很高。且內建按鍵精靈，是玩 Android 手機遊戲的神兵利器！

KOPLAYER 跑遊戲最順暢，遇到想玩的手機遊戲跑起來會卡時，通常可以解決問題。可能的例外是，這款模擬器只針對主流電腦配備最佳化，如果你的電腦配備不是主流廠牌零組件，遊戲跑起來可能更慢。這軟體本身並沒有中文版，不過 Android 作業系統可以設定為繁體中文，影響不大。

或許是受到上述兩款模擬器的刺激，老牌 Andorid 模擬器 BlueStacks 也推出專門用來執行的版本：BlueStacks Gaming Platform。它集合上面兩者的優點，既輕快、又穩定，而操作介面雖然跟 Android 手機落差很大，需要重新摸索一下才會使用，但其實反而比手機介面更簡單好用。缺點是第一次執行，要設定 Google 帳號時，反應非常非常慢，沒耐性的人可能會以為當機了。此外，還必須切換引擎模式為性能優先，預設的相容性優先並不適合玩遊戲。
