Windows 版本沿革：從 Windows 歷版的介紹來認識這套作業系統。

Windows 主要以簡便的桌面環境為優勢，讓使用者即使不具備電腦知識，也能從圖形介面一邊摸索一邊掌握到電腦的操作方式。這項特性降低了人們使用電腦的門檻，不僅成為 PC 中最普及的作業系統，更進一步帶動 PC 的發展，是 PC 產業能如此蓬勃的重要關鍵。

本章節藉由 Windows 作業系統的版本沿革來更了解它。

Windows 3.1

這是早期 Microsoft 推出的圖形介面作業環境，雖然之前還有 Windows 1.0、Windows 2.0，但到了 Windows 3.0 制定標準的圖形架構並且提供整合驅動程式的功能才算真正實用的視窗，其中以成熟的 Windows 3.1 最有名。

但是 Windows 3.1 並不算是名符其實的視窗作業系統，應該說是專為 MS-DOS 打造的視窗作業環境才對。它必須安裝於 MS-DOS 的來執行，這有點像是 MS-DOS 底下的應用軟體。

Windows 95

第一套不用依附 DOS，能夠獨立安裝使用的視窗作業系統。它主要特性如下：

１、使用 32 位元架構的系統運算。
２、支援多工作業，可同時執行多個程式。
３、直接提供中文使用環境。 (以前的 MS-DOS 還要另外安裝倚天中文才能辨識中文)

Windows 95 提供操作簡單又使用便利的桌面環境，並且提供全中文的環境，讓使用者打開電腦就能自行摸索與嘗試電腦的使用，大幅降低電腦的學習門檻。Windows 95 的推出，讓更多人願意使用 PC，堪稱 PC 史上最重要的里程。

然而 Windows 95 並不穩定，程式執行過程常常發生當機的狀況。更嚴重的是禁不起當機，整個系統會當壞掉，需要修復或重新安裝。[1]

Windows NT 4.0

Windows NT 是 Windows New Technology 的簡稱，這是一套網路作業系統，它有以下特性：

１、提供與 Windows 95 一樣親切的視窗桌面環境。
２、可以架設伺服器、搭建區域網路，並且提供網路管理的各項機制。
３、屬於全新打造的作業系統，不用與 MS-DOS 相容，因此執行上相當穩定。

當然，許多 MS-DOS 相容的應用軟體在 Windows NT 無法執行，使得它不適合做為個人工作站。

但 Windows NT 確實是相當優異的作業系統，它可以有效為公司內部搭建網路系統，讓內部的資料可以連線傳遞，而穩定的桌面讓 Microsoft Office 表現得更順暢，是非常優異的商務處理作業系統。

Windows 98

屬於 Windows 95 的下一版視窗作業系統，其主要特性如下：

１、將視窗與瀏覽器結合，提升視窗的操控性。
２、系統的穩定性有大幅的改善。
３、支援 FAT32 檔案系統，使邏輯硬碟可大於 2 GB。
４、支援隨插即用的介面卡，以及 AGP 和 USB 的支援。
５、提供 PWS (Personal Web Server) 伺服器系統，可搭建 ASP 動態網頁的網站！

將視窗與瀏覽器結合後來為人所詬病，因為 Microsoft 的 IE 瀏覽器所支援強大外掛機制帶來漏洞，使得怪客能夠透過瀏覽器惡意植入不當功能，連帶使得視窗的執行都產生影響。

在系統的穩定性方面，其實不能說真的有多穩定，只能說非常耐操，禁得起當機而已。其中以 Windows 98 SE 最耐操，幾乎不會當到壞掉，因此到現在還是有相當多人覺得 Windows 98 SE 是最好用的視窗作業系統。SE 是 Second Edition 的簡稱，第二版的意思。

無論如何，Windows 98 確實是已經發展成熟的個人桌面平台，耐操的系統架構搭配豐富的套裝軟體，適合個人工作者從事影像編輯、工業製圖、廣告排版、網頁設計、程式開發…等等使用。

Windows 2000

為 Windows NT 的下一版視窗作業系統。從下列簡單歸納出的兩點特性，更能清楚看出 Windows 2000 不凡的優異性：

１、先進的伺服器系統。
２、優越的個人桌面平台。

在先進的伺服器系統方面，Windows 2000 是在網路技術幾近成熟的時代所推出，因此涵蓋當下主流的網路技術，所提供的網路架構比 Windows NT 更能直接帶來強大的效益。例如支援 Web Application 架構，以及執行有效的安全運算機制。

在優越的個人桌面平台方面，Windows 2000 相容了 Windows 98 架構下的系統程式，因此能執行多數 Windows 98 底下的套裝應用軟體。這使得 Windows 2000 成為比 Windows 98 更可行的個人桌面平台，因此成為大專院校生最喜愛的作業系統。

Windows 2000 能滿足企業架站需求，也能滿足個人工作使用，是一套近乎完美的作業系統。但如果硬要挑的話，那就是 Windows 2000 屬於企業用軟體，價格昂貴，因此儘管它可做為當時最優秀的個人桌面平台，但一般人可能沒機會合法使用它。[2]

Windows ME

Windows ME 是 Windows Millennium Edition 的簡稱，視窗作業系統千禧年版的意思。

Windows ME 屬於 Windows 98 的下一版視窗作業系統，它不同於 Windows 95/98 的訴求在於，Windows ME 希望成為家用視窗作業系統，讓電腦能很方便地應用在生活中，成為提高生活趣味與品質的居家配備；而不像以往電腦只是商用的設備或學術的機器。

因此 Windows ME 對於多媒體應用的支援更為豐富與便利，不管是相片印表機、掃描器、數位相機的使用，或者上網、整理相片、管理文件資料，都有較為貼心的設計：「貼心而已，並沒有真的比較方便。」

然而穩定性與耐操度並沒有比 Windows 98 SE 出色，不能算是一個優秀的作業系統。

Windows XP

Windows XP 是 Windows Experience 的簡稱，視窗作業系統體驗版的意思，Microsoft 冀望這版的 Windows 能帶給使用者美好的感受。

Windows XP 是以 Windows 2000 系統核心為基礎所打造的家用視窗作業系統，取代表現不佳的 Windows ME 家用作業系統。因此 Windows XP 並不是 Windows 95/98/ME 的下一版作業系統，而是一個新的沿革才對。

Windows XP 主要特性如下：

１、使用 Windows 2000 的系統基礎。
２、使用新的視窗風貌。
３、內建數位媒體的軟體，例如數位相機、掃描器、燒錄機…等，Windows XP 都內建軟體可以使用。
４、支援更豐富的影音視聽功能，例如 Windows Media Player 撥放軟體、Windows Movie Maker 影像編輯。
５、提供系統還原、防火牆、工作管理員、遠端遙控…等系統功能，甚至 IIS 伺服器系統。

Windows XP 是非常優異的作業系統，主要在於它有非常穩定的桌面環境，支援豐富的應用軟體，並且能迎合許多網路平台的技術。因此使用 Windows XP 作業系統的電腦，可以多元性地跨足居家娛樂、商務作業、伺服系統、學術實驗…等各項領域；你可以很容易將電腦當作遊戲機，也可以很充分將電腦應用在商務處理。

然而 Windows XP 一個非常大的缺陷就是系統漏洞，這在第二版的增益套件有提供主動性的安全的機制來彌補，但無論效果如何，都已使得 Windows XP 無法安全延伸到重要商務領域。[3]

Windows Server 2003

結合 .NET 的視窗網路作業系統，是目前 Microsoft 旗下最先進的伺服器解決方案。

從目前 1.1 版的 .NET 技術來看，雖然這是有效的網路平台技術，但它在定位上並不明朗，因此結合 .NET 的 Windows Server 2003 是否值得投入並不全然是正面的答案。[4]

Windows x64

64 位元的 Windows 作業系統。雖然官方強調它不是「32 位元 → 64 位元」這段時期的過渡產品，但是……(露出狐疑的神情)

Windows Vista

代號 Longhorn 的 Windows，為 Microsoft 公司重新打造的作業系統。

由於這套新的視窗作業系統歷經多年都還打造不出來，因此各界對它都有種高深莫測的期待：「這可能是非常強大的視窗作業系統！」確實，Longhorn 預計要供應的機能真的很值得期待，像是「從資料庫型的硬碟格式導入搜尋」這類原理的技術功能。

但隨著幾項預定的機能取消，並且在 Microsoft 公司正式將 Longhorn 命名為 Windows Vista 時，就知道它最大訴求可能就是畫面長得更好看而已[5][6]。

更好看可能是指 3D 視窗翻轉和半透明視窗這類效果，但其實不然！這次用來打造 Windows Vista 的 WinFX API 並不只是單純把視窗圖形顯示而已，而是可以透過「管理」的手法讓你與系統核心訊息溝通，這可以讓開發出來的程式更直接與系統機能結合起來使用，打造出更容易發揮作業系統整體機能的應用程式來。這會讓 Windows Vista 以後的軟體用起來更順手，因為很輕易就能透過視窗達成你想要的系統功能。

此外 Windows Vista 還有一個主要優點就是能夠與更多的硬體裝置配合得很好，尤其是傷殘人士使用的輸出入裝置。我覺得這點很重要！這就像 DOS 時代大家都自稱英文白痴而記不起指令，有了 Windows 95 之後讓大家都能上手一樣，如果有了 Windows Vista 能讓功能障礙人士都能上手的話，那稱之為又一次電腦時代的革新也不為過，畢竟功能障礙人士中不乏技術能力強的人才。我相當期盼 Windows Vista 能在這方面有突破！

至於另外一項被不斷強調的「安全性」，我個人認為的確「夠主動」，但依然「不夠聰明」。智慧型安全防護的機制才是協助網管人員進一步判斷情狀的趨勢，也就是應該涉及更廣泛流程範圍的安全防護才是更好的「治標」方法。[7]

最後結論是，Widnows Vista 並不是有什麼重大系統技術革新的作業系統，它只是讓現代電腦設備得以更加順手操作的作業系統，依然稱不上我所希冀的 Windows.NET。不過，目前最需要的確實就是這樣的 Windows 作業系統！[8]

Windows 的未來

我曾經要在 Java 與 .NET 中做一個選擇。我想從 Visual Basic 程式設計跳脫出去，成為一個更高級的程式設計師。後來因為我買了一本叫做《ASP.NET 網頁設計實務》的書，裡面附贈四片光碟的 Microsoft Visual Studio.NET Beta 2 套裝軟體，那時我興奮的不得了，選擇了 .NET 做為我程式生涯的第二春。[9]

當時 .NET 的定位並不明朗，流傳四處的風聲聽起來，.NET 是用來與 Java 競爭市場的技術平台。於是我抱著 .NET 是比 Java 更先進的程式技術來嘗試，可是從 .NET 過於先進的願景身上，我開始認為 .NET 並不是用來與 Java 競爭的技術平台，而是一個新解決方案的核心。而這新解決方案不是指別的，就是 Windows。

.NET 的願景是讓應用程式能夠在主動式網路服務的架構下來運行，讓應用程式能夠隨時因應新的資訊科技更新系統核心，並能在主動式網路服務的環境下互相回饋系統資訊。這種過於先進的願景必須同時有作業系統能完善地結合才有辦法實現，因此這立刻可以讓人警覺到，.NET 並不是先進的 Java，而是新式的 Windows。

換句話說，.NET 是新式 Windows 的程式引擎，與其說它是程式技術平台，不如說它是一種用來建造那種新式 Windows 的函式庫會更貼切，就像現在 Windows 95/98/ME 的 WIN32 函式庫那樣。

如果真是這樣，那未來的 Windows 將成為空前卓越的作業系統！例如你不用透過 C 語言才能修改 Windows 的系統核心，而是透過 .NET 在安全與易用的保證下自行增益 Windows 核心，而且這樣 Microsoft 也能在不開放原始碼的商業犧牲下，就達到公開安全的需求。

.NET 的特性可以幫助延伸 Windows 的未來！甚至終結 Windows 的未來：推出 .NET 作業系統。因此 Windows 的未來與 .NET 息息相關。
