Notepad++ 設定集

平常設計程式的話，用 Visual Studio Code 比較有效率，自動完成、語法提示、執行、除錯、版本控制、檔案總管…通通在一個軟體完成，不用在工作列轉來轉去。

不過檢視純文字文件、以及修改檔案文字內容時，用 Notepad++ 比較直覺、簡單，它 Ctrl+H 的 Replace 還有 Ctrl+Shift+F 的 Find in Files，根本是修改純文字文件的神器，更別提應有盡有的 encoding/decoding 和 convert 等內建於功能表的項目有多方便了。單純檢視純文字文件的話，也更快速、順手，算是取代記事本用的，所以兩個都裝。

平常寫程式碼的工作交給 Visual Studio Code，日後需要大量修改每個原始碼檔案的內容時，就交給 Notepad++ 好好表現了！

Notepad++ 當然也能靠 plugins 打造成集自動完成、語法提示、執行、除錯、版本控制、檔案總管於一身的開發環境，但東拼西湊的，而且 plugins 不是它強項，很多功能想做出來難度很高，所以表現還是不如 Visual Studio Code。

右鍵選單

免安裝版沒有右鍵選單用 Notepad++ 開啟檔案的功能，必須自行下載 NppShell.dll：

http://docs.notepad-plus-plus.org/index.php/Explorer_Context_Menu

裡面有兩個檔案，差別在 32 位元和 64 位元。使用 64 位元 Windows 10 的話，將 NppShell64.dll 複製到 Notepad++ 所在資料夾。

以系統管理員身分執行命令提示字元，切換到 Notepad++ 資料夾的路徑，然後輸入：

regsvr32 /s /i NppShell64.dll

會跳出設定畫面，直接按 OK 即可。

日後要移除的話：

regsvr32 /u NppShell64.dll

檔案關聯

在 Windows 10 要讓檔案與免安裝版 Notepad++ 產生關聯，最好用功能表的 Settings → Preferences → File Association 設定。

必須以系統管理員身分執行 Notepad++ 才有作用。

Theme

在套用 Notepad++ 內建的 Obsidian 配色主題時，其深沉的背景顏色上，綴飾著外星科技感的綠色文字，讓我敲起程式碼來，有種在宇宙間傳遞訊息一般的新奇感…於是修改 Obsidian 把主要關鍵字改為綠色，並取名為「捷克隕石」的 Moldavite。

說明

適用 HTML、CSS、ECMAScript、XML、PHP、ASP、Python、Ruby、Java、C、C++、C# 語言，其它語言則保持原風格不變。修改重點，在於關鍵字統一為綠色，註解為桃紅色，屬性為亮黃色，值為橘黃色，基本資料型態為紫色，網頁內容或程式碼符號為藍色，底色為黑板綠。

安裝

解開檔案後，將 Moldavite.xml 放在 Notepad++ 安裝路徑的 themes 資料夾裡面，然後重新啟動程式。

依序點擊功能表的 Settings → Style Configurator，然後在 Select theme 的下拉式清單選擇 Moldavite 即可。

疑難

在 Windows 8 選用等寬英文字型，可能有中文亂碼的問題。建議用「細明體」，大小 12，文字閱讀起來其實很舒適。

原本 Windows 10 也有中文亂碼的問題，年度更新版後已正常，可以選用 Courier New、Consolas、DejaVu Sans Mono 之類的等寬字型，大小 11，呈現更美的閱讀環境。

7.3.3

修正 CIA Hacking！

CIA 為了蒐集情報，透過網路在你我的電腦植入間諜程式，側錄你操作特定軟體時的情況，這些軟體包括 Notepad++。

Notepad++ 在 7.3.3 版防範了這個問題，因此不建議使用 7.3.2 或更舊的版本。

當然，沒被 CIA 植入間諜程式的話，即使你用 Notepad++ 7.3.2 或更舊版本也沒問題，不用太擔心。

7.0

開始提供 64 位元版本！但不支援過去 32 位元的 plugins，所以沒在用。

6.8.9

若有其它軟體也讀取 Notepad++ 6.8.9 開啟的文件，下次再執行 Notepad++ 重新載入該文件時，會造成軟體當機。因此該版本已從官網移除，並建議使用者退回 6.8.8 版。

6.7.8

釋出三天後就緊急修正 Missing msvcr120.dll 的問題，四天後又緊急修正 WinGup error 的問題，所以這版本的下載網頁從選單中移除了。不過 Google 的話可以連進去。
