終於被 Linux 幹掉的 Windows 10

歷年說法

2004 年，我說：「1998 年推出的 Windows 98 做得比 2000 年的 Red Hat Linux 6.1 + CLE v0.9 好。」

2009 年，我說：「2001 年推出的 Windows XP 做得比 2002 年的 Mandrake Linux 8.2 好，甚至也比 2006 年的 Ubuntu 6.10 好。」

2013 年，我說：「到了 2009 年 Windows 7 還是做得比 2010 年的 Linux Mint 8 好。」

2017 年，我終於可以說：「elementary OS 0.4 幹掉 Windows 10 了！」

Windows 10 變成操作繁瑣的作業系統

為了成為跨裝置的作業系統，Windows 10 必須在 Modern UI 與 Desktop 互相搭配。

結果，以前簡單按一下就好的工作，很多要從傳統桌面的「控制台」轉跳到平板介面的「設定」，變得要多按好幾下滑鼠，操作繁瑣。

Windows 10 不再是穩健的桌面作業系統

無論 Windows XP 還是 Windows 7，桌面環境是和作業系統緊密連結在一起的，你看到的視窗怎麼點就怎麼反應、不點就不反應…這很正常不是嗎？不這樣才奇怪吧？從 Windows 8 開始就變奇怪了，因為 Deskop、Modern UI、System Core 形成三方鼎立的狀態，所以桌面環境與作業系統脫節開來了！

結果，視窗常常處於不受控制的狀態，例如彈出對話視窗時，居然被放在所有視窗最底層而看不見訊息，你必須親手點對話視窗來看。當你安裝一個軟體時，很可能因為看不到安裝完畢的對話視窗，然後一直等它安裝，但其實早已安裝完成，只是對話視窗被擺在最底層，所以看不到訊息。

相反的，會忽然有某個視窗自己跑到最頂層蓋住你正在操作的視窗，而且它就像置頂視窗一樣，不會因為你正在操作的視窗獲得駐點而疊在下面，而會一直蓋在最上面！你必須先點一下那個忽然冒出的視窗，再點一下正在操作的視窗，才會疊在下面。

這種視窗失控的問題，從 Windows 8、Windows 8.1、Windows 10、Windows 10 Anniversary Update 一直存在，似乎是先天缺陷、無法解決的問題了。連這麼簡單的視窗控制都出問題，就更別提其它 Windows 10 桌面環境的靈異現象了～

在家用電腦時代，Windows XP 是多麼棒的視窗作業系統，讓 Linux 望塵莫及。進入平板電腦時代，Windows 10 怎麼努力也不是 iOS 和 Android 的對手，卻反而把自己老本「最穩健的桌面環境」給賠掉了。反觀 Linux Mint 18 的 MATE 已經被打造成耐操桌面環境，Cinnamon 則被打造成順手的桌面環境，工作起來比 Windows 10 有效率多了！

我想，Windows 10 恐將變成用來安裝遊戲軟體用的作業系統，如果你想配置一台工作用的電腦，你會想安裝 Windows 7、Linux Mint、FreeBSD，而拒絕安裝 Windows 10，它會讓你工作效率變低落～

越到後來 Windows 10 更是變成問題多多的作業系統

Windows 10 隨著版本持續更新，已顯露出品管不佳、把使用者當測試員的問題。一次又一次低級的 BUG，已讓人對升級到新版 Windows 10 感到恐懼，深怕用好好的作業系統，哪天忽然這個也故障、那個也故障，影響日常工作。

由於 Satya Nadella 將重心放在雲端，也成功將 Microsoft 轉型為系統服務商，因此早已大幅縮減 Windows 原本引以為豪的龐大測試團隊！過去這測試團隊，組別分得很細，每一組各以自己的專業，測試屬於自己範疇的功能模組。這也是為何 Windows XP 到 Windows 7 的品質無懈可擊，Linux 再怎麼追也沒辦法打造出一樣穩健的桌面環境：「在使用者還沒遇到低級 BUG 之前，就已經抓出來修好。在使用者還沒操壞某個功能之前，就已經找出來解決問題。」當 Windows 交到使用者手上，就是令人安心的產品！即使是 Windows 8，也只是「設計上」不被喜歡，但「使用上」可是很穩健的！

這規模龐大的測試團隊縮減到近乎裁撤後，Microsoft 改用自動化測試做品管。效果如何，Windows 10 使用者經過這幾年來的體驗都很明白：「擔心死了！」以前只要是正式版就絕對不可能發生的低級 BUG，現在可以每次「被迫升級」後，就把測試版才會遇到的低級 BUG 交到使用者手上，造成使用者的困擾。

現在更是裁撤 Windows 部門，歸類於體驗與裝置部門，你還相信 Windows 10 會是以往你所認識的 Windows 嗎？我們用的到底是實驗品？還是功能穩固的產品？

適合在桌面工作取代 Windows 10 的 Linux 發行版 TOP 5

這是我在 2017 年為 Linux 發行版所做的排名，可能已經不合時宜。

Linux Mint

在 Ubuntu 穩定、耐用的程式執行環境基礎上，主打基於 GNOME 改造的更耐操桌面環境：MATE 和 Cinnamon。

使用體驗不斷與最新版 Windows 跟進，因此想在桌面工作上取代 Windows 10，非 Linux Mint 莫屬。

缺點是連系統需求都與 Windows 很像，電腦老舊一點就跑不太順 Linux Mint。

由於耐操到極點，才是桌面工作最重要的，因此排第一，若沒有專業人士需要量身打造工作環境的特殊需求，選 Linux Mint 就對了！

elementary OS

在 Ubuntu 穩定、耐用的程式執行環境基礎上，主打基於 GNOME 改造的 Pantheon 桌面環境。

對喜歡 macOS 的人，應該會覺得 elementary OS 是更好的 Linux 桌面作業系統！這奇幻美妙的桌面設計，會讓人越看越想放棄 Windows 改用 Linux，很想在這樣的桌面打造工作環境～

迷人的是，這麼美的 elementary OS 竟是輕量級作業系統，系統需求不高，操作起來非常流暢。耐操度也很強，但還不至於像 Linux Mint 堪比 Windows 7 那樣隨便操也不會把程式和系統玩壞的程度，因此排第二。但如果你不是那種喜歡亂操作一通才來怪系統不穩定的人，elementary OS 的耐操度已經令人滿意了！

elementary OS 跟其他發行版通常更新一下核心和套件、補補漏洞，剩下的沒什麼改進就推出新版不一樣，elementary OS 每一版都有明顯的優化，也經常發明新的解決方案讓自己變得更好，每次都感受到這作業系統更耐操、更流暢！照這樣下去，耐操度早晚和 Linux Mint 不相上下，卻更流暢、好看、易用，TOP 1 只是時間早晚而已吧？

Manjaro

在 Arch Linux 極簡、高效的系統基礎上，致力於為它打造人性化的使用環境。

桌面環境預設使用 Xfce，有自己的軟體庫，這發行版的情況，很像基於 Debian 的 Ubuntu，在完美中再進一步苛求，所以我想這發行版終將表現出如同 Ubuntu 般的影響力，為 Linux 發行版帶來新的境地。

然而訴諸的是為不夠人性化的 Arch Linux 提供人性化，讓更多人能受用 Arch Linux 的高效系統表現，並不是桌面工作的使用環境，因此桌面環境這部份做得沒有前兩個介紹的 Linux 發行版來得好。也不是說做得不好，而是沒有它們來得好，畢竟開發的目標不一樣。

所以這比較適合專業工作者使用，像是經常測試程式的軟體工程師、或需要耗用大量系統資源的影像處理人員。若前兩款 Linux 發行版效能低落、無法滿足你的需求，往往可以在 Manjaro 得到滿足，因此排第三。

Lubuntu

在 Ubuntu 穩定、耐用的程式執行環境基礎上，主打節省系統資源的 LXDE 桌面環境，並將應用軟體相依性降到最低，成為順暢、易用、耐操的作業系統。

缺點是稍嫌簡陋了些，沒有華麗的桌面特效、也沒有先進的系統功能。但如果電腦買來當文書機，或像我一樣打打程式、上上網、頂多玩玩模擬器遊戲，那這速度快又耐操且不容易壞的作業系統，會是不錯的選擇！

更別說電腦太舊跑不動其他 Linux 發行版就只能跑 Lubuntu 看看了 XDDD

總之，這是我排第四的桌面工作用 Lunux 發行版！雖然這是個很無趣的作業系統，但適合訴諸實用的人。像我用的就是 Lubuntu，我不喜歡把明明很快的電腦，耗用系統資源在無意義的聲光特效上讓作業環境變慢，乾淨俐落才是我要的。

Fedora

原 Red Hat Linux 的發行版，目的並不是作為桌面工作的作業系統，而是系統功能卓越的作業系統，工作上比較符合老派 UNIX 的應用精神，因此獲得 Linus Torvalds 認可，並使用這個發行版。

以 Linux 發行版來說，這是比上述發行版更卓越的作業系統！適合架站、佈署主從架構、或當 UNIX 工作站使用。但對桌面工作來說，東西太多，塞了一堆桌面工作者用不到的東西，有點大材小用了！雖然也不是說不能當桌面工作的作業系統，但這比較像電腦工程師用的桌面作業系統，因此排在最後。

附帶一提，還有基於 Red Hat Enterprise Linux 的 CentOS，但除非真的需要架一個企業級的伺服器，不然就像 Windows Server 2016 一樣，桌面工作者還是用 Windows 10 就好～

並不是最強的作業系統就一定好用，適不適合才是好用的關鍵。換句話說也不是視窗桌面就一定好用，有些工作場合像是收銀台或機電整合，把操作寫死在一目了然的簡陋介面或用指令下動作反而好用，在層層收納的圖形界面翻來找去只是整死自己。

Windows 10 真有那麼爛？

也不是，Windows 10 持續增加新功能，其實越來越好。那些功能說真的，大多是 Linux 沒有的！光是在 Xbox 遊戲列下的功夫與用心，就足以羨煞 Linux 使用者。

問題就出在只顧增加新功能，品管卻變差，容易改版出有問題的作業系統來。

在上面 Linux 發行版排名中，我重視的是耐操度，功能和美觀是其次。但 Windows 10 正好往功能和美觀邁進，卻問題多多沒做好品質管理，成為品質不穩定的作業系統。做為一個需要用錢買、而且至少 4000 元的產品，這是非常要不得的！做為商業公司，品管是攸關倒閉的問題，有重大缺失就亟需改進。但現在卻一副這樣就好、無關緊要的姿態，我已對 Windows 感到擔憂，彷彿看到當年 IE 擺爛終將兵敗如山倒、市佔率一輸就再也救不回來的歷史要重演了～

我只想要一個別出問題，能讓我安心工作的桌面環境。但使用 Windows 10 的我，永遠不知道被迫升級後、甚至被迫更新後，原本用好好的功能就這樣故障，影響我好幾天的電腦作息。反而在 Lubuntu、Linux Mint、elementary OS 我不需要擔心這些，一來這些發行版功能上已經穩定下來，不太新增有的沒的導致系統變糟，二來即使發展新功能，雖然同樣測試與品管的人力不足，卻步調很慢的進行，像 LXQt 花了四年才正式取代 LXDE，不會倉促的把低級 BUG 放進正式版搞得使用者好像免費測試員一樣。

有意義嗎？

即使我說 Linux 終於幹掉自爆的 Windows 10，其實也沒什麼意義，個人電腦裝載的作業系統依然是 Windows 市佔率最高，而且是絕對性優勢！Linux 則是低得可憐，連人家零頭都不到。依 2018 年分析結果，個人電腦使用 Windows 的人有 83%、macOS 有 13%、Linux 只有 2%，這麼一看，你會想開發 Windows 的應用軟體？還是沒人用的 Linux？

好不好用不是選擇的唯一條件，市佔率往往更關鍵！除非 Windows 10 真的爛到連市佔率都輸給 Linux，否則沒意義。因此你不需要看到我說 elementary OS 比 Windows 10 好，而棄用、鄙視 Windows 10：「Windows 10 只是自爆，但還沒爛掉！」

上上網、寫寫程式，那 elementary OS、Linux Mint、Lubuntu 都比 Windows 10 順手、好用！但需要更多用途，像是電玩遊戲、平面設計、工業製圖…其他稀奇古怪的玩意兒，那依然是市佔率高、吸引絕大多數人開發軟體的 Windows 10 來得完備。因此你不需要換，自己用得開心就好。

這世上大概只有我們寫程式的人會用得不開心吧？因為我們電腦買來不是視聽娛樂用的，而是能動就好：「能跑我們寫的程式就好。」我們甚至電腦配備回到二十年前也沒差，照樣能從中找到「操作電腦」的樂子。聽我們這群怪咖說 elementary OS 幹掉 Windows 10 了，而放棄 Windows 改裝 Linux 來用，我想你不會覺得開心，因為每個人的樂子不同。我們找到的樂子對大多數人來說，其實一點也不有趣，像在 Mandriva Linux 跑 NetBeans IDE 的樂子、在 elementary OS 跑 Visual Studio Code 的樂子…你能體會的話，才適合放棄 Windows 改裝 Linux 來用，否則繼續裝 Windows 10 跑 Path of Exile 會比較有趣～

[*] 引進智能學習發展自動化測試降低不關心的業務成本確實是不錯的想法，但還不完備時，不應該且戰且走的用這套自動化測試發展你的主力產品。趕快砸錢把品管穩固回來，等智能學習發展的自動化測試完備了再省這筆錢吧！別成功創造了新的事業帶領 Microsoft 走向巔峰，卻在你手上把 Windows 搞砸了，最後毀譽參半，那就太可惜了～
