Regular Expressions（正規式）

regular 是「有規則可循的」，將這個規則性找出來，再套用到字串比對有沒有符合規則的資料，就是 regular expression。

中文是從字面上看不出什麼意思的「正規表示法」「正則表達式」，但功能就是：表達某種規律形式的詞句，以比對出符合這種規律形式的資料來使用。

語法

符號

/ /正規式要寫在這符號裡面
.任意字元
-字元範圍
[]字元集合
()字串
|或
^開頭（出現在 [] 內表示否定）
$結尾
?0 或 1 次
*0 或多次
+1 或多次
{,}幾次

跳脫字元

\eEscape
\nNew line
\rEnter
\tTab
\fForm feed
\000 到 \777以八進位值表示字元
\x00 到 \xFF以十六進位表示字元
\符號正規式用到符號
\d相當於 [0-9]
\D相當於 [^0-9]
\w相當於 [0-9a-zA-Z_]
\W相當於 [^0-9a-zA-Z_]
\s相當於 [\f\n\r\t]
\S相當於 [^\f\n\r\t]

選項

在 /正規式/ 後面，可以設定如下參數：

g匹配所有資料（不因匹配到資料就停止匹配其它資料）
i不分大小寫
m匹配多行資料
u匹配 Unicode 字元

實例

/ABC/ABC
/^ABC/以 ABC 開頭
/ABC$/以 ABC 結尾
/A.C/A + 任意字元 + C
/ABC|DEF/ABC 或 DEF
/[ABC]/A 或 B 或 C
/[(ABC)]/ABC
/A[XYZ]C/AXC AYC AZC
/A[X-Z]C/AXC AYC AZC
/A[^XYZ]C/A + XYZ 以外的字元 + C
/A{3}/AAA
/A{2,4}/AA AAA AAAA
/A{2,}/最少兩個 A
/A{,4}/最多四個 A
/AB{3}C/ABBBC
/A.{3}C/A + 任意三個字元 + C
/A[XYZ]{3}C/A + 任意三個 X 或 Y 或 Z + C
/AB?C/AC 或 ABC
/AB+C/A + 多個 B + C
/AB*C/AC 或 A + 多個 B + C
/A/iA 或 a

[^否定條件] 或 [^\S] 可做為萬用字元。

應用

篩選圖檔名稱

底下 JavaScript 程式只會顯示符合條件的 image.jpg 和 image.gif，其它不會顯示：

搜尋 Google Picasa 圖檔網址，取代為相對路徑。

在支援 Regular Expressions 的 Notepad++ 中（需勾選「類型表式」），「搜尋目標」設為 (http)[\S]*(/20)，「取代成」設為 ../../figures/overcast/20，則：

<img src='https://lh4.googleusercontent.com/-Z0i6ueEk-II/Tk9lJEBk_RI/AAAAAAAAAkg/5dQKz4jinuU/s640/20110715A_01.jpg' alt='插圖'/>
將變成：

<img src='../../figures/overcast/20110715A_01.jpg' alt='插圖'/>
