Linux = 穩定 ???

2004 年，我報讀職訓中心全額補助的電腦組裝課程。上到安裝作業系統這一課，講師提到 Linux 的種種優點。不過我說 Linux 的桌面環境不穩定時，講師很訝異地質疑說：「不穩定？怎麼可能，Linux 耶？」

是的，很不穩定。

獨自運作很穩定，不表示讓人操作就很穩定。

大家對 Linux 的印象就是這個廣告說辭：「穩定的作業系統，持續運作一年也不會當機。」

沒錯！但這是指 Linux 當主機的表現！

如果電腦只是放在機房運作，那裝載 Linux 確實有非常穩定的表現。多少人會因為免費的 Linux 居然比貴死人的 Windows Server 耐操、有效率，而成為 LAMP 陣營[1] 的追隨者！

不過，如果電腦是拿來上網路、打報告、玩遊戲、看 VCD、以及其它工作用的軟體，說實話，Linux 很不穩。因為做這些事的時候，我們是在 KDE 或 Gnome 這專為 Linux 設計的視窗桌面環境，所以穩不穩定看的是 KDE 與 Gnome，而不是 Linux。

Linux 很穩定沒錯，但 KDE 與 Gnome 並不穩定，你會發現在 Linux 的桌面工作其實很容易當機。

Linux 只是提供視窗桌面環境而已，並不是視窗作業系統。

這也難怪，現在 Linux 的視窗桌面，讓我想到 Windows 3.1 的視窗桌面環境：「注意！我不是稱視窗作業系統！」

Windows 3.1 必須在 MS-DOS 下安裝與執行，可說只是安裝 Windows 3.1 的操作介面來控制 MS-DOS 作業系統而已。儘管 Windows 3.1 讓人驚艷，還是擺脫不了 MS-DOS 操作上的限制。直到 Windows 95 擺脫 MS-DOS，獨立成為一個作業系統，才正式為 PC 使用者帶來一個穩定、有效率的桌面。

Linux 並不是一個「視窗作業系統」，它只是提供「視窗桌面環境」而已。[2]

事實上 Windows XP 比 Linux 提供更穩定的桌面環境。

回歸主題，雖然 Linux 是穩定的系統，但不是只要跟 Linux 扯上關係就代表什麼都很穩定。或許可以這麼說：「Linux 是拿來跑的，但不適合拿來用。」不能因為 Linux 用來架設伺服器有穩定的表現，就連帶宣稱 Linux 的桌面也很穩定。

真正穩定的桌面環境是 Windows XP 才對！在 Windows XP 執行程式，已經不會再像 Windows 95/98/ME 三不五時當掉，程式在執行上不僅更有效率，操作起來也極為順手。更重要的是，Windows XP 很難弄壞，以前 Windows 98 大概每三個月就系統損毀要重灌，Windows XP 用一年半了還不重灌，也能保持穩定的桌面環境。

當然，也不是說 Linux 的桌面環境不適合上網、聽音樂，使用 KDE 或 GNOME 來做這些事情還是很便利！但我不能否認 Windows XP 在這方面做得更好。[3]

[1] Linux + Apache + MySQL + PHP。
[2] 但也不是「KDE 何時才能擺脫 Linux 成為獨立的作業系統？」的問題，我認為問題出在沒有一個可公認為標準的桌面環境被推舉出來。大家在開發 Linux 視窗應用軟體時，連桌面環境就不同調了，穩定性當然有問題。（編按：寫這篇文章時，Ubuntu 還沒誕生。）
[3] 當我誇獎 Windows XP 作業系統時，應該會有很多人拿「系統漏洞」反駁 Windows XP 並不是好的作業系統。系統漏洞造成的安全性問題確實是 Windows XP 一大缺失，不過這表示 Windows XP 不是優良的網路用戶端作業系統，跟本文宣稱的 Windows XP 是穩定的桌面作業系統，屬於不同層次的議題。如果安裝 Windows XP 電腦並未連接網際網路，系統漏洞的問題就無從影響 Windows XP 的使用品質。
