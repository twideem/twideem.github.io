家用電腦的時代看 Linux

「我覺得 Linux 作業系統必須正視的問題是：Linux 不適合家用電腦。」

從專科學校肄業那年開始支持與嘗試 Linux 作業系統到現在，轉眼已經八年了。這些年來，在大家看到 Linux 作業系統的桌面越來越穩定之際，我卻看到它落後家用作業系統的幅度越來越大！

「桌面!?」坦白講，這已經是過時的話題！

都已經 2009 年，現在看一套作業系統好不好用，必須從「家用電腦」的角度，而不是從「職用電腦」的角度。現在的電腦，不再是買來做為文書處理的吃飯工具，而是轉為家庭娛樂的電器。一款作業系統的實用性，不再是「桌面環境跑 Office 順不順手」，而是「能不能一次相容所有的電腦週邊電子產品」，就算不能，起碼也能「以同樣一種手法簡單處理完成」。

2009 年 Windows 已經是成熟的家用作業系統，不管是老媽還是小孩，想在 Windows 作業系統處理軟硬體問題，經常只要不停按「下一步」就可以搞定。反觀 Linux 作業系統，卻還是在「穩定、便利的桌面」的話題上面在打轉，對於幫助使用者更直覺處理軟硬體相容性問題的人性化設計，沒有用心整頓，就連表現最好的 Ubuntu，一旦碰到硬體相容性問題，一陣手忙腳亂後通常還是束手無策收場。除非你本來就熟練 Linux 的系統管理才會覺得得心應手，但這就不符合家用作業系統的期待了。

Linux 作業系統奮戰史

Linux 最早是做為免費的 UNIX-like 作業系統，用來架設「主機」有著運作穩定、效能優異的表現。加上它以 GPL 發布，對一直缺乏作業系統的 GUN Project 就像救世主，成了開放原始碼的共主，積聚大量愛好者。

在電腦因為 Windows 95 普及化後，Linux 陣營也開始為提高普及率而努力！

1998 年納入 KDE 桌面的 Linux 發行版 Mandrake 算是分際點，讓 Linux 陣營士氣大振，正式與其它主流作業系統競爭版圖。受這股潮流感化而來的「移民」，更鼓吹改用 Linux 就能辦到 Windows 的事。顯然這群人電腦買來是打打報告、寫寫程式用的，不曉得電腦正朝結合更多娛樂功能的方向前進。

然而，Linux 的桌面真的很穩定很好用嗎？事實上在 Ubuntu 發行版本成熟之前，從來就沒穩定過。Linux 穩定的是「系統本身」，但「桌面」只是「應用軟體」不是「系統本身」，而 Gnome 和 KDE 一來程式本身設計得不夠精實，二來各家 Linux 發行版本只是把這些桌面環境加裝上去了事，並沒有進一步優化這些桌面環境的實際運作狀況，因此 Linux 的桌面一直都不穩定。是到 2004 年 Ubuntu 發行版本的出現，才開始強調桌面環境的整合度。

Linux 的應用軟體也不足，撇開致命的驅動程式不談，連辦公室軟體都是到了 2002 年才出現一統江湖的 OpenOffice.org，在那之前等於沒有辦公室軟體可言。

家用電腦的時代

在 Linux 企圖以桌面和辦公室軟體與主流作業系統競逐市場時，主要對手 Windows 作業系統的版圖早已從商用跨足到家用。

注意！這正是當今 Linux 需要正視的問題所在！現在是家用電腦的時代，那個「電腦沒裝 Office 還稱得上是電腦嗎？不能工作簡直跟廢鐵一樣。」的時代，不知不覺已經過去。今天電腦買來可能是為了連網際網路、為了玩線上遊戲、或者為了數位相機，作業系統扮演的角色變成結合家電的應用中心，如果電腦不能像家電一樣提高居家品質，才會像廢鐵一樣！

但多媒體娛樂這一塊 Linux 會打得比 Office 一戰更艱辛！因為 Linux 必須堅持「開放原始碼」這一原則，而提供顯示卡晶片的廠商也有將驅動程式視為商業機密的原則，兩者難有妥協的空間。

另外，Linux 至今無法推出可與 DirectX 相比的多媒體開發平台，想在 Linux 發揮多媒體設備最高表現，讓家用電腦展現更大的娛樂效果，目前做不到。

你可能會說有 OpenGL，它只是提供 2D 圖形與 3D 模型的功能，這對遊戲開發來說只是「基本功能」，如果不能像 DirectX 可以直接存取顯示卡的資源，就不能發揮顯示卡本身最炫麗的 3D 特效，變成這些特效只能靠 OpenGL 用軟體方式產生，不僅麻煩，而且拖累電腦的速度。

Linux 振作吧

每個人都不希望每隔四、五年，就得付費給 Microsoft 才能讓自己這台寶貝電腦跑最新的作業系統，我也是如此，所以很希望 Linux 能在家用電腦有所表現。

不過殘酷的現實就是，想讓電腦有更好的娛樂效果，最佳的選擇還是 Windows！就算是有口皆碑的 Ubuntu，跟 Windows 比起來也還早得很，家用娛樂電腦可不是放個 MP3、播個 DVD 就叫精采，能否發揮電腦硬體最大的效能，讓廠商得以迅速創造無限可能，才能讓電腦融入居家生活時更加精采。這點對發展緩慢的 Linux 卻是致命傷。

作業系統的選擇其實很簡單：「如果新買來的電腦已經搭載 Windows 作業系統，那就不用再考慮 Linux。」「只有在你的舊電腦想跑新作業系統，才考慮用 Linux 來脫胎換骨。」

但我還是期待 Linux 有一天成為表現優異的家用作業系統，這樣才能讓我撿便宜。

[*] 2015 年再看這篇 2009 年寫的文章，Linux 不見得沒機會在家用電腦上與 Windows 競爭。如果所有的軟體服務都由網際網路提供，那用 Linux 或 Windows 的差異就不會太大，而在這 HTML5 的時代，各種網頁平台的影音函式庫，搭配 asm.js 或 WebAssembly，我想軟體開發商差不多可以大展身手了。
