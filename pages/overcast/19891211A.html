我幹過的蠢事

記事本

中學時期，老是忘記學校交待的事項，於是買了小小一本行事曆，寫下這些天要做的事。

但一回到家就忘了看行事曆 Orz

想說這套電腦遊戲可以玩一輩子，不買來收藏嗎？

1998 年的某日，下午只有兩堂課就放學的我，到草屯市區逛電腦遊戲專賣店。看到清倉拍賣的運動類遊戲，像是籃球、足球、棒球，想說現在買來，以後想玩場球賽就不用愁了！

這想法不錯，那時候的遊戲都是 3D 製作，雖然畫面不真實，但遊戲場景是立體的，以運動類遊戲來說玩起來還是很有感覺。

2014 年世界盃足球賽開打，一股熱想到這套遊戲，拿出來玩時…別說玩了，連安裝都不能安裝。

這時覺得當年的想法蠢死了～

詭異的英文名稱

還在念書時，英文很破，認得的單字不多，手上也沒有中翻英的工具，所以使用電腦時，自訂了一堆奇怪的術語，像是：

Bedou（系統程式）、Bvst（備份）、Comexe（執行檔）、Drun（捷徑）、Gex（破解工具）、Hork（專案）、Inuset（需安裝的軟體）、Kibtor[1]、Mbrsee（說明檔）、Mund（音樂）、Obray（文書）、Picolor（圖片）、Ubate（上傳）、Wbsrv（網站）、Yoller（遊戲）。

現在看了，真為當初的自己感到丟臉～

更奇特的是，這套術語居然還有名稱：Nihom…有空發明這堆名詞，還不如好好背單字 Orz

曾經抗拒 1024x768，堅持 800x600 螢幕解析度。

都 2001 年了，還在用 166MHz 處理器電腦，顯示卡也只有 2MB 記憶體的我，都是在 800x600 解析度的螢幕工作～

在我買 1400MHz 處理器、RIVA TNT2 32MB 顯示卡的新電腦，嘗試 1024x768 螢幕解析度時，發現針對 800x600 設計的網頁會走樣，就產生排斥感。

我調回 800x600，自我安慰現在還很少人用 1024x768，鄙視那些沒事用什麼 1024x768 讓字變小不舒服的人。

沒幾天開始發現，解析度變大，桌面可以裝更多視窗，讓工作更方便，「頓悟」之後也改用 1024x768 了～

副檔名很長

剛出社會，投入進階電腦技能的那個時期，我用很長的副檔名，像是：

我的文件.document（*.doc）
我的圖片.picture（*.bmp）
我的腳本.script（*.js）
我的樣式.style（*.css）
我的組態.resource（*.ini）

現在看了，真想問十年前的自己腦子到底在想什麼～

提出 Linux.JET 方案與大談 .NET 的願景

我要從 Visual Basic 轉攻 JBuilder 時，Microsoft 剛推出 .NET Framework，因此猶豫該學 C# 好還是 Java 好～

當時沒人知道 .NET 究竟是什麼：「事實上連 Microsoft 自己也不知道怎樣解釋它是什麼。」不少人討論的不是 .NET 程式怎麼開發，而是它是什麼？哪裡比 Java 卓越？

只花 269 元買一本《ASP.NET 網頁設計實務》，裡面就內附三張 Visual Studio .NET Beta 2 中文版安裝光碟的我，像撿到寶似的開心安裝使用，於是我先選擇 C# 而不是 Java。用的時候，我覺得 .NET Framework 是類似 WIN32API 讓 Windows 適合程式開發者打造成網路作業系統的 SDK，因此我認為接下來會有 Windows.NET 這樣新型態的作業系統出現！

但既然是網路作業平台，.NET 會淪為必須與 Windows 綁在一起才能發揮最大效應的方案。試過 SharpDevelop 之類組合，看能不能擺脫 Microsoft 的控制，答案是不可能，於是我提出 Linux + Java 的解決方案，稱為 Linux.JET，正式投入 Linux 與 JDK 的陣營！

JET 是 Java Environment Touchable 的縮寫，意思是：「可觸的 Java 環境。」因為 JDK 對「使用者」並不友善，只靠「開發者」推展不了 Linux.JET 概念的網路作業系統。

由於這個想法一直沒有實現，現在回想起來蠻「蠢真」的。後來智慧型手機的 Android 作業系統很符合我當初 Linux.JET 的概念，但還少了 server to server 的概念，依然只是 client per server 的被動式網路作業系統，網路作業系統應該更主動提出自己的服務，才能真正締造 Web 3.0 的時代，讓整個網際網路成為主記憶體那樣，等著你的作業系統去循環裡面的資源。

對了，.NET Framework 是什麼？四、五年後，看到版本演進到 .NET Framework 3.x 了，還是很平常無奇的 SDK 而已，才明白這只是慣於破壞標準、好將天底下所有軟體都綁死在自己旗下 Windows 才能執行、以此做為最大利益的 Microsoft，用來替代 Java 的方案。當年 Windows 客製化專屬自己的 Java VM 時，很有可能 Java 的規格演變成 Microsoft 說了算，導致 Java 程式不再是「Write once, run anywhere.」，於是 Sun Microsystems 提出控告並且取得勝訴。但 Microsoft 是真的需要有類似 JVM 這樣的架構，將被動式單機作業系統的 Windows 與提供主動式網路作業系統的平台「隔開」來「銜接」，於是推出自己的 .NET Framework （相當於 JVM）與 C# 程式語言（相當於 Java），企圖在 Enterprise web services 這塊領域也搞 Windows only，因為這塊正由 Java 勝出。難以啟齒的 Microsoft 因此很難解釋 .NET 是什麼 XDDD

於是不只 Linux.JET 的想法很蠢，當年大談 .NET 願景也是我幹過的蠢事[2]～

搜尋

奇摩與 Yahoo! 合併後，我發現搜尋的精確度越來越低，因為從網名搜尋變成資料比對，所以常常找不到網站，都連進一些掛羊頭賣狗肉的網頁。

於是我改用網擎 Openfind，並為此沾沾自喜，好像自己很有見識一樣。

但當時已經有 Google 了，我卻連聽都沒聽過 Orz

1366x768

16:9 寬螢幕流行時，我納悶買這種把桌面拉寬變形的東西有什麼好處？

後來才知道有 1366x768 這樣的螢幕解析度，可以讓桌面開更多應用程式視窗，而且桌面加寬所帶來的方便，要比 800x600 提升到 1024x768 更顯著～

不過，在我買 16:9 寬螢幕之前，一直都不知道這件事，以為那種東西只是把 1024x768 拉寬而已～

對 Chrome 的設計反感

Chrome 剛推出時，不但把各家網頁瀏覽器那一長串的按鈕精簡到剩兩三個，還把頁籤放在最上頭，多餘視窗標題列去除，簡潔的操作介面感覺平易近人，非常劃時代的設計！

但切換頁籤是網頁瀏覽器最頻繁使用的功能，把頁籤放在離頁面較遠的位置，我得更費力移動滑鼠去切換網頁，所以起初我很抗拒 Chorome，繼續吹捧 Firefox 3.5。

自從 Firefox 4 也跟進這樣的設計，漸漸習慣後，現在倒是用得很開心。偶爾回頭下載 SeaMonkey 來玩，看到頁籤緊黏在網頁上面的設計，整個擁擠感，不自覺皺起眉頭來：「當初堅持這種設計才好的我，真是有夠蠢的。」

而且不是普通的蠢！我本想拒用跟進這項設計的 Firefox 4，還曾發文批評說 Firefox 是在自我邊緣化，很快就會被 Chrome 10 和 IE 9 消滅。也因為既然跟進別人的設計了，那還不如改用 Chrome…是的，我最早使用 Chorme 的契機，就是因為 Firefox 4 跟著把頁籤放在最上頭。

電腦專家？

我以為懂程式設計，就可以當專家，時常在 ZDNet、PCADV 電腦王（Ｔ客邦）、癮科技…這些網站裝內行，大放厥詞～

好幾年後才發覺，3C 電子科技的專業性，高於程式設計，我過去其實都在做自曝其短的事 Orz

我那時四處嘴砲的暱稱有「與妻訣別書」和「比克大魔王」，現在 Google 的話，還是可以找到不少那些可笑的回文 XDDD

[*] 台灣 ZDNet 倒了，Ｔ客邦我申請刪除帳戶了，癮科技我幾乎將所有留言刪除了，只有引用我回文的內容還留著。

去冰

平常到茶飲店總是這麼買：「大杯珍珠奶茶兩杯，微糖，去冰。」不想讓冰塊侵占奶茶的空間！

同樣的，便利商店第一次推冰的現煮咖啡，我去買時這麼說：「兩杯冰咖啡，不加冰塊。」

結果店員跑去問領班該怎麼辦，因為超商的冰咖啡是現煮後再用一整包冰塊讓熱騰騰的咖啡變冰，我當場就跟傲客一樣 XDDD

不過領班服務態很好，並沒有把我當刁難的傲客，還想辦法說會幫我把冰塊降到最低，努力滿足我的需求。但我當時真的不知道冰咖啡的做法，不是故意刁難，就是有夠蠢而已～

只看搞笑劇、把電腦圖書當散文來讀

我有人格缺陷，無法看悲劇類型的戲劇，只能看喜劇，否則會有種心靈受創、難以承受的感覺。

像《3x3 Eyes》這我有預感男女主角聚少離多、恐怕以天人兩隔收場的故事，我國中時（1995 年）只買前兩集，接下來的劇情我逃避不敢看。2014 年才「鼓起勇氣」整套看完，而且幸好有「外傳」，描述八雲與三隻眼過著幸福的生活，否則我恐怕會後悔看這部漫畫。

我都看「蠟筆小新」「周星馳系列」這類對多數人來說沒水準的搞笑劇。

因為我能看的戲劇不多，所以很少看漫畫和小說[4]，平常想看書時都拿「電腦圖書」當散文讀，偶爾想換口味時就拿「日語教材」或「遊戲攻略」來看 Orz

一個人有沒有出息，看他人格健全度是能探知一二的！像我這種禁不起挫敗、只想逃避往安樂地方跑的人，今天淪為沒出息的失敗者，很理所當然。

另外我喜歡看日劇，不過蠢的是劇情演什麼不重要，因為我只是想看女主角或女配角那美美的樣子而已：和久井映見（1996）、戶田菜穗（2002）、矢田亞希子（2003）、長谷川京子（2005）、上戶彩（2013）、真野恵里菜（2016）。會因為劇情關係吸引我看的，是「庶務二課」「王牌大律師」這種日劇～

反韓

我非常厭惡南韓這個瞧不起台灣的國家，偏偏台灣超哈韓，第四台越來越多整天都在播韓劇的頻道！

我能做的就是把那些頻道都刪了！現在按 20 下遙控器的下個頻道，就可以從「CH18 國家地理頻道」轉台到「CH58 非凡新聞」，其中電視購物就佔了四分之一…而且賣胸罩時蠻好看的。

反蘋果

我買程式設計的電腦書籍，不管作者多有名，只要在圖中看他用的是 OS X，就整個反感。

一個高竿的程式設計師，怎麼會選擇在封閉平台、阻礙競爭的蘋果電腦寫程式？我不認為從這種人身上能學到什麼，這種書我丟掉好幾本了！

然而，在行動智慧的時代，高竿的程式設計師們將近 3/4 用蘋果電腦，因為 iOS 是比 Windows 8/10 熱門的觸控手勢平台，適合用來開發 iOS App 的 OS X 是 Unix-like 的作業系統，不在 OS X 開發程式，而在連新版 Safari 都沒有的 Windows 寫程式，是頭殼壞去嗎？

所以我又幹蠢事了！那些能讓我學到東西的專家，當然用 OS X！

重回 Google 陣營

由於 Chrome 飆速 JavaScript 提升 Web 2.0 時代，以及 Google 旗下有各種免費又好康的服務，我成為 Google 品牌迷信者！

直到 2011 年，看到 Google 以綁死自己旗下產品為基準來「增、刪」Chrome 功能，而不是使用者需要與否，這自稱不為惡卻很偽善的面目，我開始對 Google 品牌反感。不只如此，使用者經常在 Google+ 與 Facebook 角力下被犧牲，不是關閉服務、就是改變操作介面、甚至強迫 YouTube 綁死 Google+ 帳號。

對 Google 品牌反感後，就不再用 Chrome，連 Gmail 也不想看到，於是在 Outlook.com 出現，試用覺得滿意，便轉投 Microsoft 陣營。

2014 年，我架個人網站的 Webs.com 開始限制免費用戶的功能，只好另尋網頁空間。Microsoft 的 SkyDrive 不支援 web hosting，Pancake + Dropbox 無法讀取 *.ico、*.7z、*.ogg、*.jar，最後選擇 Google Drive。既然登入帳號了，那信箱就改回 Gmail，又重回 Google 陣營了～

但我蠢的不是先前改用 Microsoft 服務，而是我又繼續用 Google 的品牌。2016 年，Google Drive 要關閉 host webpage 了，我又要轉移他處了 XDDD

這已經是 Google 關閉的第 ∞ 個「還有很多使用者在用」的服務了！如果沒有永續經營的打算，就不要為了在市場分杯羹推出產品，讓願意永續經營「旗下只有這項產品」的公司好好為大家服務。因為結束一項產品會影響到使用者的權益，一次兩次三次還無所謂，你已經Ｎ次了，漸漸成為一家對支持者沒有誠信的公司，以後誰還敢用你這家公司的服務？只是出來亂的，卻有著響噹噹的公司招牌，阻礙其他只靠特定產品的公司發展。

無論如何，我已經不再使用 Google 品牌的服務，我嗅出這家公司腐敗的味道～

重回 Microsoft 陣營

重回 Google 陣營是很愚蠢的事，但重回 Microsoft 也是很愚蠢的事。

自從 Outlook.com 改版成為 Micrsoft 雲端產品之一後，操作介面反應變得非常慢。點進郵件了，卻遲遲不顯示內容，要等快十秒才看得到。不然就是按鈕怎麼按都沒動作，想寫個新郵件一直按 + 鈕都沒反應，就像被當白痴耍一樣。想到隔壁有速度快到隨點隨動的 Gmail，卻堅持用慢到跟軟體故障沒兩樣的 Outlook.com，實在蠢到極點。同樣的情況也發生在 OneDirve，不管做什麼事，都要等五秒左右，開個純文字文件，要等八秒。明明有速度快到不用等的 Dropbox，卻選擇慢到跟磁碟壞軌沒兩樣的 OneDrive，我實在蠢到極點。

明明知道 Microsft 產品特色就是慢，還重回 Microsoft 陣營，結果整天就是等等等…等它慢慢處理，已經蠢到不曉得該怎麼形容，我懷疑自己智商是不是低於 70！（很有可能）

但我很喜歡 Satya Nadella 以擁抱開放的方式領導 Microsoft，將一家惡名昭彰的企業轉變成貢獻自己專業能力的公司，所以我依然選擇 Outlook.com，然後改用 Windows 10 內建的 App 來收發解決網頁版 Outlook.com 太慢的問題。我知道重回 Mirosoft 陣營是很蠢的是，但我嗅出 Mircosoft 懂得創新的氣息，所以我要搶在大家還在敵視、戲謔 Microsoft 時使用這品牌的服務。

[*] Outlook.com 在 2017 年改版後，變得超順暢了。

沒有社交帳號

我不喜歡在滿滿都是白目、嘴砲、正義魔人的網際網路與人交流，所以我沒在用 Facebook 或 Twitter 這類社交網站。甚至即時通熱到發燙時，我沒 Yahoo!奇摩即時通也沒 MSN Messenger 帳號。

本來有用遊戲基地或 T 客邦之類的帳號，在討論區或留言串與人交流。但互看不順眼的人越來越多，所以在個資法規定網站必須提供會員刪除帳號的途徑後，我就把帳號刪除了。

但網路搜尋的資料，有些是在 Facebook，必須登入才能順利閱讀，我也只能申請 FB 帳號。不過剛剛說過我沒在用，所以整個活動紀錄都是空的。

附帶一提，我也沒有手機號碼，因為我在現實世界也不喜歡和人交流。我交流的對象只有電腦。想聯絡我的話，只能透過電子郵件：twideem@outlook.com，不過換過好幾次電子信箱，過去認識的人其實聯繫不到我，能蠢到這地步，可說是神人了。

不會「盜版軟體」反而電腦功力退步

不用盜版軟體，持續研究有哪些免費軟體可用，已經快十年了。

後來覺得，我不會 Alcohol 120%、CloneCD、AnyDVD、威力導演…反而「電腦功力」退步了！（也不會 Sliverlight、Adobe Edge Animate、SharePoint，我只會 HTML5。）

只是我寧可「不會電腦」，也不想使用下載的盜版軟體去做那些事情。我依然把對電腦的興趣，放在蒐集免費軟體，能做多少事算多少；能夠敲敲程式碼、架架伺服器就夠了～

或許我 34 歲不年輕、人生態勢抵定，對電腦逐夢不再熱情，所以過去是電腦能學多少就學多少，多學一種，未來就增加一分希望；現在是能做多少就做多少，差那一件事對人生沒什麼影響～

認真

轉眼 39 歲了！回顧過去整理的蠢事，覺得那反而是我人生精彩的一部分 XDDD

還有其他蠢事嗎？這些年來，對新的遊戲大作沒興趣，覺得只玩回憶中好玩的舊遊戲就夠了！就像不追時下最新的流行歌曲，只聽年輕時喜歡的那些音樂一樣。即使想來點不一樣的，也是找過去年代我沒接觸過的～

程式設計也不再追新技術，覺得把需求定義降低到、把問題變得單純，再用最笨拙的方式去完成，也是可行的方法。

漸漸覺得，過去認真看待每一件事，就是在幹蠢事了。

握筆的方式

我從小握筆的方式，是三隻手指一起抓著筆，就像爪一樣。

雖然我知道這握法不標準、跟別人不一樣，但平常畫漫畫習慣這樣拿筆，改了的話畫不好，就一直沒改。

大概三十歲的時候，早已沒在畫漫畫了，所以有一天想說找不到事做，就嘗試中指托住筆，只用拇指和食指夾住筆的標準握法。一開始不習慣，寫久了倒也自然，字還比以前漂亮了一點。

不過，寫字寫久時，食指的指甲碰到中指的肉會痛，卻一直沒在意。快四十歲的時候，才覺得不對，用小時候三隻手指一起抓著筆的握法寫看看，發現沒這問題。原來我當初還是握錯了，食指要再伸出來一點，避開食指指甲與中指皮膚的接觸。於是我又重新學習握筆的方式，練好的字跡又走樣了…等等！重新練過前，先用網路再查一下，發現還是錯的，筆不能靠虎口，也不是用夾的，是拇指和食指捏在一起再把筆放下面就好！幸好發現得早，不然到五十歲又重新學握筆的方式就搞笑了 XDDD

本站以 Vivaldi 最佳化版面配置

在 Chromium 瀏覽器壟斷市場的時代，我想針對 Firefox 最佳化版面配置，為反壟斷盡一份心力：「如果 Chromium 成為唯一，受害的就是使用者自己！」

但 Firefox 對本地端網頁實在不友善！滑鼠手勢不能開啟 file:// 的超鏈結，新增分頁也不能指向 file:// 的網頁，這對在本機端藉由網頁平台設計 RIA 的我，等於提早受 Firefox 殘害，不用等到 Chromium 壟斷市場只剩一種網頁瀏覽器。

Mozilla 其實是一家固執己見的組織，完全以自己的意識形態設計 Firefox 網頁瀏覽器，不讓使用者決定是否要開啟哪些核心功能、關閉哪些核心功能，逕自決定使用者該怎麼做，這跟令人擔憂的「壟斷後一家獨大的廠商」不是同一德性嗎？

所以，我只好選擇基於 Chromium 打造的 Vivaldi 來用，它人性化數十倍，我可以放手打造本機端的 RIA 來用，不用為了迎合 Firefox 把自己放在硬碟的網頁架在 HTTP 伺服器透過 localhost 讀取，因為這反而增添安全性的風險。

明知用 Chromium 是蠢事，卻因為用 Firefox 對我來說跟壟斷後一家獨大、自以為是的自大廠商一樣受害，不得不蠢下去，真是悲哀，卻又無奈～

時代明明越來越進步，可用的網頁瀏覽器卻比 2000 到 2010 年還要少，只剩 Chromium 和 Firefox 兩種可用，真不知所謂「進步」是怎麼個進步法？在倒退的路上很進一步向前是吧！

唯一可以確定的是，正在壟斷網頁瀏覽器市場的 Google 與 Chromium 開放原始碼組織，反而比市佔率低到只有 9% 的 Mozilla 更了解使用者需求、更尊重使用者權益。

面對這情形，我也只能安慰自己：「就讓 Google 壟斷吧！」別為 Mozilla 這種只顧堅持自己主張、沒意願去配合使用者的組織擔憂那麼多。
