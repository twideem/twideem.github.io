Linux Mandrake 發行版創辦人：Gaël Duval

讓 Linux 發行版開始與 Windows 較勁的 Mandrake

1998 年，一名 25 歲的法國人 Gaël Duval 率先挖掘 KDE 桌面環境的潛在價值，將 KDE 包裝於 Redhat Linux 5.0，推出 Linux-Mandrake 5.1。它不僅擁有華麗的桌面環境，而且安裝也相當簡單，簡直是最接近 Windows 95 作業系統的 Linux，一舉引爆 Linux 作業系統的桌面熱潮，推出不到兩個禮拜就大受歡迎。這年 Gaël Duval 順勢成立 MandrakeSoft 公司。

此後，Mandrake Linux 以發展桌面市場為目標，除了搶鮮加入最新的桌面環境與應用軟體，並持續開發容易操作的視窗介面程式，讓使用者能夠方便又能穩固地管理 Linux 各項系統設定，大幅降低 Linux 的難度[1]。這些努力讓 Linux 展現親和力，簡便到完全不懂 Linux 的人也能無痛安裝[2]，而且對 Linux 支持者來說，有了 Mandrake Linux 對推廣 Linux 變得無往不利，Mandrake 對 Linux 的貢獻是非常巨大的！

如此風光的 MandrakeSoft 曾於 2001 年在巴黎的 Marche Libre Exchange 上市，卻因公司決策錯誤，意圖轉型為網上教育供應商，導致每月支出過高，而且難以負荷，從此影響公司的財力。往後營運陷入困難，甚至 2004 年一度瀕臨破產。

2005 年，MandrakeSoft 與拉丁美洲最大的 Linux 廠商 Conectiva 達成併購協議，成立新公司 Mandriva，而 Mandrake Linux 也更名為 Mandriva Linux。[3]

2006 年 Mandriva 再度虧損嚴重，連身為共同創辦人的 Gaël Duval 都遭公司解僱。

後來 Gaël Duval 投身 Ulteo 計畫，開發線上桌面系統，希望能將應用軟體藉以移植到網路平台上。

沒有 Gaël Duval 的 Mandriva 果然不是 Mandrake

我第一次接觸的 Linux 發行版是 Red Hat Linux。當時在台灣 Red Hat 才是第一選擇，普遍認為 Mandrake Linux 在伺服器部分明顯不如 Red Hat，而 Red Hat 預設的 Gnome 桌面環境也很便利，所以 Mandrake Linux 只能排老二。在那 Linux 是用來架站的時代，也只好如此了！

不過我很快就迷戀上比 Red Hat Linux 更適合桌面應用的 Mandrake Linux，甚至幾年後在國外忽然冒出打下知名度的 Ubuntu，我也裝來玩了一下，還是繼續選擇支持 Mandriva Linux。

當時我很不滿意 Ubuntu 的安裝緩慢、以及不靈活 (no root) 的帳戶管理機制，不過我對它「不只精簡、更求紮實」的作業環境相當「介意」，比起新舊系統混雜的 Mandriva Linux，Ubuntu 的環境經過嚴密的整合，顯得它很穩固、耐操。「既然要做為桌面環境，穩固耐操應該比華麗取巧來得好吧？」不免有這樣的猶疑。[4]

不過沒有了 Gaël Duval，我對 Mandriva 便不再有信賴感，甚至開始懷疑 Mandriva 再也不會是 Mandrake。果然，Mandriva 為了振興事業，開始從「市場面」來發展，想藉由個人桌面的基礎來吃企業客戶端 (Client) 這一塊看看，事實也證明 Mandriva Linux 很少再進步，只是不停重新包裝上市罷了，連最基本的「新鮮感」都沒了！

這次沒什麼掙扎，就投入 Ubuntu 的懷抱。
