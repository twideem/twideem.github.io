Shift、Ctrl、Alt 意義上的區別

Shift（切換鍵）

切換鍵盤本身的功能。

例如將 a 切換為 A。

這是最早出現、最基本的打字機組合鍵。

Control（控制鍵）

執行系統功能或調用外部指令，這些比較像是控制電腦用的，而不是應用軟體本身的功能。

例如 Ctrl+C 複製、Ctrl+Z 復原。

Ctrl 的組合鍵經常通用於其它應用軟體，同樣的按法，功能大同小異。

Alternate（替代鍵）

將應用軟體內部的功能改用 alt 組合鍵執行。

例如 Alt+F 展開功能表的 File（檔案）、 Alt+E 展開功能表的 Edit（編輯），把原本用滑鼠點來執行的功能，改為鍵盤也能使用。

Alt 的組合鍵通常是應用軟體為自己量身訂做，同樣的按法，在不同軟體會有不一樣的作用。

[*] 這是早期 DOS 時代圖形使用者介面的區別，例如 PE2 或 QBasic 那種很簡陋的圖形使用者介面，一看就知道該用 Ctrl 還是 Alt。隨著 Windows 作業系統推出二十多年，視窗應用程式的圖形使用者介面設計得越來越複雜，Ctrl、Alt、Shift 早已混用，甚至已有新的意義上區別，例如主要功能用 Ctrl，不夠用換 Alt，再不夠用就 Ctrl + Alt，相反功能用 Shift 或 Ctrl + Shift～
