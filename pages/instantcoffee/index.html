爪哇即溶咖啡

取名自咖啡豆的關係，大家常常用咖啡的意境看待 Java 程式語言，基本上確實有相符之處：

１、因為振奮精神而舒緩疲勞。
２、咖啡的苦滋味，可以道地品嘗原香，也可以調配奶糖讓它香醇。
３、人類社會最廣泛流行的飲料。

關於第一點，其實 Java 並不會讓你在最快時間內解決程式問題，所以寫 Java 程式會讓人疲憊。不過當你用 Java 解決問題時，這次成功經驗能直接解決其他同樣問題，你會覺得這次疲勞是有意義的。

關於第二點，其實 Java 是很難用的，因為它提供的類別庫相當低階，所以不夠人性化；你必須具備相當程度的專業知識才會覺得順手，無法直接光看字面上的意義了解該怎麼用，用起來是很辛苦的。但懂得品嘗的人明白那是為了保持物件導向的彈性，有這種彈性才能活用到所有問題。而不喜歡這種彈性的人，可以加入他人準備好的工具組，或者自己動手讓情況變成你想要的，照樣能用得順手。

至於第三點，Java 是世界上最強悍的程式語言，卻免費讓大家下載使用[1]，目前擁有最高使用率。

也因此，在我想要寫一本針對一般電腦使用者、而不是程式設計師的 Java 程式入門書籍時，便想到《爪哇即溶咖啡》這個書名，有「隨手就能沖泡一杯 Java 咖啡」的意思。這本書會嘗試以精簡幹練的內容，直接將想要的功能表現出來，免得浪費太多時間在閱讀上面，把功能盡快實作出來才是重點，也才稱得上是「即溶咖啡」。

經過近十年的研發，我的即溶咖啡還是不怎麼好泡，幸好有英文網站早已符合即溶咖啡的需求：Java programming notes。

在 Oracle 收購 Sun Microsystems 時，我一度對 Java 的未來擔憂，因此本站變成透過 Java 來介紹「物件導向」的專欄，不大介紹怎麼使用 Java 語言本身…很不幸這麼做是對的。

．jEdit - 雖然操作介面不夠方便，但編輯性能堪稱第一的文字編輯器。
．Ant - 將編譯、測試、部署工作自動化的專案管理工具。
．JUnit - 可實現「測試先行開發模式」的單元測試框架。
．Struts - 符合 MVC 設計精神來開發 Web Application 的 Framework。
．Sprint - 全方位應用程式框架，可建立輕量級的 EJB 應用程式。
．Hibernate - 能將物件的關係，對照為關聯式資料的開發框架，更有效結合資料庫。
．Tomcat - 可執行 JSP 的 HTTP 伺服器系統。
．Jakarta EE - 企業級 Java 應用程式架構。
．GlassFish - 企業級 Java 應用程式伺服環境。
．JDOM - 比 Java API 簡單易用的 XML 解析器。

．XOM - 能將 Java 物件序列化為 XML 模型。
．SWT - IBM 以 Widget 規格為 Java 開發的圖形使用者介面，比 Swing 進步。
