電腦技能的學習

什麼是電腦技能？電腦裡面的所有軟體通通要會操作，才算是電腦技能嗎？

我看到資料處理科系的課程，可說是什麼軟體都學，不管是文書處理、美工繪圖、資料庫系統、程式開發工具…幾乎企業界有用到的軟體，資料處理科系的人通通要學，被操得快要像唸設計系的一樣。

不過我遇到唸資料處理科的網友，總是謙虛地跟我說：「其實什麼都學不精。」

說到學習電腦技能，我認為不需要每樣都學。電腦是可以在各行各業都有用途的設備，只要在自己有興趣的行業，去學習怎樣使用電腦來推動它，這就是符合專業層級的電腦技能。

相反的，如果你對各行各業都沒興趣，只想說學會電腦來幫每個行業做事就好，我想學習這樣的職業技能並非長遠之技。畢竟，電腦裡面裝的應用軟體越來越容易操作使用，可能大家都學得會，所以如果你的專業只是操作這些軟體，而不是在某個工作領域的專業，想靠電腦吃飯會很危險吧？

正確的專業心態

電腦是一套全方位的設備，它能夠讓各種業務性質的公司展開工作，使得電腦成為職場上一個重要的項目。因為如此，電腦成為許多人積極學習的一項技能，希望能夠因為會用電腦而找到好的工作。

但是，學習電腦有時會落入一種迷思：「以為電腦才是主要專業」。

事實上，學習電腦技能，建議不要認為你是在學電腦，因為正確的心態應該是「學習使用電腦來完成某項專業」才對。我們應該是學習怎樣使用電腦來做印刷排版、怎樣使用電腦來做工業製圖、或者怎樣使用電腦來做室內建築。儘管當初學的是電腦，但專業領域還是應該放在印刷排版、工業製圖、室內建築等方面。

這是為什麼呢？因為電腦只是輔助你完成某項專業領域的工具。別忘了，我們不斷看到電腦裡面的應用軟體越來越容易操作，所以只學會這種不會很難學的輔助工具，而在主要領域不專業，想起來就知道會很慘。除非你想從事的是電腦組裝維修，你的專業領域才是電腦。

Windows 作業系統是最好的例子！自從有了 Windows 之後，我們週遭朋友或同學，有誰是先上過電腦補習班才會使用電腦的？我們發現，有了 Windows，讓我們打開電腦以後，什麼都不懂也能自己慢慢摸索，很快就只到怎樣使用電腦來寫報告和上網了。

電腦技能也是，印刷排版學習使用 Illustrator，工業製圖學習使用 AutoCAD，網頁設計學習使用 Dreamweaver，程式設計學習使用 Visual Studio……這些應用軟體也是越來越好學了，每個人都可以學得會。所以，如果你認為電腦技能只是學會使用這些裝在電腦裡面的軟體，而不是這些軟體所要工作的領域上求專業，你覺得你有競爭力嗎？你不怕很快就被淘汰嗎？

我曾經待過一家機械工廠，我的老闆跟我說過，他以前讓學電腦的學生接案，要用 AutoCAD 為工廠設計機械製圖，結果這些沒幹過工廠的人，交出來的機械製圖實在不符業界規格。懂機械製圖的人再學電腦機械製圖比較符合專業。

程式設計最好？

說到電腦技能，程式設計總是在人們心目中排得高高在上的。有種程式設計才是電腦方面最優越的技術，其它的電腦技能像是文書處理或美工繪圖就比較沒那麼了不起……這恐怕是一種時代落後的想法了！

如果在以前，程式設計確實不是簡單的技術領域。當時要投入程式設計領域，應該要對電腦這個機械設備有一定程度的認識，例如中央處理器與記憶體對於資料運算與存取的基本運行原理，因為他們必須按照電腦運行的特性來安排程式該怎麼走，不懂電腦怎麼動，就無法下達命令讓電腦按人的意思運作。所以這批人對於整個電腦設備的運行原理相當了解，對於電腦在運作上發生的各種問題，心中多少都有一定概念知道是怎麼回事了！

現在呢？自從電腦完全進入 Windows 系統的時代後，程式設計常常是以某種方便工具來開發，例如 Visual Basic，它讓事情變成只要懂 Windows 就好，電腦這種機械設備是怎樣運行的根本不是什麼問題。因為，電腦的運行都已委由 Windows 處理，不管中央處理器還是記憶體，都是 Windows 以自己的方式在管理的，於是變成只要按照 Windows 特性去設計程式即可，電腦本身的運作原理已經不重要。

所以囉，現在說他是學程式設計的人，他真的懂多少電腦的原理嗎？他懂的可能跟你差不了多少吧。而且現在程式設計很多根本也是一套 Windows 的應用軟體而已，這跟使用文書處理軟體或者美工軟體的人還不是一樣！把程式設計跟電腦專家混為一談的時代已經過去了。

我想說的是，學習電腦技能沒有說哪種比較優越，重要的是所學的電腦技能是否能提升你專業領域的工作效率，能達到這個效益才叫優越！

也不是說程式設計不好，只是說你要挑你用得到的，或者你有興趣的。如果只認為程式設計比較優越，所以挑這個電腦技能來學，那恐怕只是虛榮，不見得會比學習其它電腦技能的人有好的出路。

例如，電腦技能方面，我覺得印刷反而是更好的選擇！它可能就包含廣告設計與印刷工業兩種專業領域，可以藉由累積數年經驗來提升你不輕易被人取代的能耐，雖然電腦化，也只是流程上的輔助作業而已，不影響你的專業。在電腦人人會用，卻沒甚麼專業領域的時代，類似這樣的工作才有保證。所以你的興趣在哪，就朝哪邊發展你的專業，想在社會上有更好的出路，電腦會成為你的輔助工具，但別以為電腦就是你的專業。

搭配學習延伸的電腦技能

對正準備想學習電腦技能以投入職場的社會新鮮人，我個人的建議是，除了電腦技能，你還要專攻延伸出來的輔助性技能會更好！

例如，你想學習程式設計，那建議在學程式語言時，可以專攻軟體工程來做為你的專長。軟體工程是指對程式開發案作分析、發展與測試的工作，聽起來就是用來加強、輔助電腦程式設計工程的業務，屬於電腦程式設計所額外延伸出來的相關事業。讓這種延伸出來的輔助性產業成為你的主要專業技能是明智的做法，它將讓你立刻比同個職場上的夥伴顯得出類拔萃！

在我們剛剛準備投入職場時，都認為學會產業的主要專長是最重要的競爭優勢。不過仔細想想的話，你會發現這是最劣勢的競爭條件。因為大家都會專攻主要技能，而每個人的專業都將達到快要極限的程度，所以後來不管你怎麼拼都不會跟別人有很大的拉距。這時，反而次要的技能是主要專長的人，將立刻扮演提升主要工程業務之開發效率的領導角色！

再以電腦程式設計為例，投入這行業之後，大家都會專攻程式語言，每個人都在拼語法技巧，拼到後來大家都很強，就算有高下之分，其實彼此程度也差不了多少，都是高手、專家。這時，如果你是唯一具備「系統開發管理」的專業人才，那你馬上就顯得比其他人更能有效提升程式設計之產業效率。

不只電腦程式設計，像學習 Dreamweaver 的人應該把美術設計或廣告版面當自己最強的專業，也是同樣的道理。

電腦的學習門檻將會越來越低，想在電腦業界保障你的工作並提升你的地位，具備延伸的專業知識、相關的產業技能，是最佳的出路。

所謂電腦白痴

2006 年了，有時候還是會聽到週遭的人自稱「電腦白痴」…嗯，我想現在已經沒有電腦白痴了，最起碼你不是會打字、會上網、會玩遊戲嗎？

所謂電腦白痴是指電腦開機後完全不知道怎麼動手操作，只能呆在螢幕面前，感覺像智障一樣。

這種現象只會發生在 DOS 作業系統的時代，因為那種作業系統必須以輸入指令的方式來操作電腦，而剛開始接觸電腦的人當然不懂任何一個指令，只能愣在黑色的螢幕面前，覺得自己現在好像白痴一樣。

現在沒有電腦白痴了吧

到了 Windows 作業系統的時代，我想不該再有人自稱電腦白痴了。

當生平第一次開機，進入了 Windows 作業系統，將會看到一個美麗的圖形畫面。隨著所看到的精美畫面，你會在感覺新奇的心情下嘗試順手移動看看滑鼠，然後左按右按試個幾次，開始有點搞清楚滑鼠可以怎樣點選畫面上的東西。你會一邊擔心不知道怎樣會把視窗給弄壞，一邊刺激的在視窗畫面中找東西來按，你可能按到叫做 Internet Explorer 的圖案，然後發現自己進入「Yahoo!奇摩」的網頁……

你覺得你有愣在螢幕面前無法反應嗎？

在 Windows 作業系統的時代，整個佈滿圖形的畫面讓你就像在玩電動玩具一樣，可以一邊探索、一邊學習。你不會說完全不知道該從何動作起，視窗作業系統提供的圖形畫面世界讓你有能力在四處各地來回冒險。

確實，你所需要學的還很多，但你絕對沒有像個白痴，愣在電腦面前無法反應。

電腦白痴是 DOS 時代的名詞，到了 Windows 的時代，你頂多只是「剛開始學」而已，但你絕對不是電腦白痴。

Java 1.5 新增功能介紹（一）老虎咖啡

Java 1.5 因為大舉提升語法機能而備受期待，為了表達語法威力的提升，還刻意取為 Tiger，表示這次的 Java 就像老虎一樣勇猛。

Java 1.5 新增的語法功能如下：

1. Generics - 能讓集合類別進行型別檢查。
2. Enhanced for Loospan - 可直接取出陣列結構資料的迴圈。
3. Autoboxing/Unboxing - 提供區間自動轉型的功能。
4. Typesafe Enums - 一般程式設計的列舉型態。
5. Varargs - 讓參數的數量是可變動的，不再拘限於固定個參數。
6. Static Import - 用來事先宣示類別關係，讓物件內部功能的呼叫變得簡化。
7. Annotations (Metadata) - 利用程式碼的注解製作原始碼樣板。

以上 1 到 6 項新功能將在本台陸續介紹，至於第 7 項功能，我把 Java 官方說明檔讀了好幾次還是看不太懂，所以就不做介紹了。

當然這只是語法上新增的功能，系統所提供能功能還多著呢！有待我們幾番玩味。

Java 1.5 新增功能介紹（四）Autoboxing/Unboxing 自動裝箱處理

Autoboxing/Unboxing 叫做「自動裝箱與反裝箱」。

什麼叫做「裝箱」呢？把基本型態轉成物件型態就叫做「裝箱」，例如 int 轉為 Integer。相反的，物件型態轉為基本型態就是「反裝箱」，例如 Integer 轉為 int。

以前像這種基本型態與物件型態之間轉換的工作，是必須由程式設計師自行撰寫程式來處理的，而到了 Java SDK 1.5 Tiger，編譯器將自動處理基本型態與物件型態之間的資料。

由於這是編譯器自動執行的工作，所以並沒有實際語法是用來表示封箱處理的。只要你想把基本型態的資料交給物件型態，或者顛倒過來，就會自動進行封箱處理，妥善讓資料在兩者結構中轉移。

Java 1.5 新增功能介紹（五）Typesafe Enums 型別安全列舉

Typesafe Enums 叫做「型別安全列舉值」。吼！這什麼鳥名字？有夠冗長。

好啦，馬上來看「型別安全列舉值」的白話文，就是：將某個功能可能使用的設定值，列舉出來，讓程式設計師遵循套用，避免程式設計師隨意輸入資料。程式設計師因為使用列舉值，所以輸入的資料都符合該功能結構的需求，讓功能在執行時不會使用到錯誤的值。
