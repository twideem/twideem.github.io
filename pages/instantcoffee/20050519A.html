final

如果你覺得 final 應該不是什麼了不起的東西，現在趕快來看看它在你設計 class 時，會帶來很大的實用性。

final 資料欄位

將 field 宣告為 final，表示它設定了初始值後，就不能再設定新的值給它，等於 constant（常數）。如果試圖在原始碼中修改資料，將無法通過編譯。


值得注意的是！並不是在宣告的時候就一定要同時給予初始值！

先宣告為 final，然後執行一段程式運算，再將計算好的結果，在 constructor（建構式）中設定給它，既可動態賦予內容，又能防止被修改，是很棒的機制！

將這種資料欄位設為 public，對開發者來說，設計感變得更充滿彈性，而不是死板板的只能開放 method 來操作 object；畢竟！有些時候就是覺得 field 比較直覺，例如 Math.PI 其實就是 public static final double PI。

final 資料欄位未給定初始值的話，編譯器會告知錯誤，所以可以放手去用這項設計手法。

final 操作方式

將 method 宣告為 final，表示禁止 override（覆寫）。


這項機制可以確保特定 method 只能照你所想要的設計來運作。

當你不希望 class 被人家透過繼承的方式給 override 掉，產生預期外的行為表現，就可以使用這項功能來限制。

例如驗證密碼的動作，很可能被 override 後，以永遠傳回 true 的方式來規避你的設計。

final 類別

將 class 宣告為 final，表示不能使用 extends 來繼承。


java.lang.Math 就是裡面的數學函式跟數學符號都希望寫死，所以無法被 extends 的例子！

很自然的，既然 class 無法被繼承，那就沒有 method 會被 override 的問題，所以 fianl 的 class 裡面 method 都是 final，或者說不需要再宣告為 final。

但 field 還是可以修改的，並未預設為 final，這點需要留意。
