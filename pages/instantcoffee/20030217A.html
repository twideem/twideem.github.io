亂數

速成亂數做法

如果沒有特殊需求的話，建議直接用 Java API 的 java.util.Random 來產生亂數。

0 到範圍內


從 0 到 2 之間，隨機產生一個數值。
你可能會感到疑惑說，預設的計算結果並不直覺，為何設定 3，並不是 0 到 3 或 1 到 3，偏偏是 0 到 2？

是不直覺沒錯，但這樣有個好處是，能直接處理「陣列」或「群集」的資料，因為這些結構在資料長度為 3 的時候，卻是從 0 到 2 去存取。可能設計 API 的人認為這種情況居多，所以預設這樣的值吧？

自己變通一下，把得到數值加 1，即為 1 到 3 之間的亂數了。

最小值到最大值

true 或 false

java.util.Random 的 nextBoolean() 可以隨機產生 true 或 false。

傳統亂數做法

因為通常程式語言都會提供 Math 函式庫，讓我們使用 round() 與 random() 來產生亂數，所以很多人還是習慣用 java.lang.Math 來設計程式。

0 到最大值


從 0 到 10 之間，隨機產生一個數值。
使用 random() 產生帶有小數的亂數，再用四捨五入的 round() 搭配 (int) 轉為整數。

1 到最大值


從 1 到 10 之間，隨機產生一個數值。
使用 random() 產生帶有小數的亂數，再用無條件進位的 ceil() 搭配 (int) 轉為整數。

0 到最大值-1
