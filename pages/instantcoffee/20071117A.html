JSP 舊文

JSP 的範例教學多次改寫，這裡放的是過去式文章。

JSP 簡單易用的初衷

從 ASP 說起

二十世紀 90 年代尾聲，為了推廣「動態網頁」，Microsoft 推出了 ASP (Active Server Pages)，結果大受歡迎！

ASP 可將「伺服端程式碼」寫在 <% %> 當作「標籤」混在 HTML 裡面，這讓我們設計「使用者與伺服器網站互動」的「動態網頁」時，可以直接用 HTML 的寫法來表現，然後找適當的位置插入「希望伺服器端運算的程式」即可！不但大幅降低伺服端程式設計的難度，而且 ASP 不需要 IDE 這類程式開發工具，它寫起來很簡單，就像在寫 HTML 原始碼一樣，有效提昇開發動態網頁的效率！

而且當時並非只有 Windows NT 這類「伺服器專用視窗作業系統」內建 IIS（Internet Information Server）能執行 ASP，連 Windows 98 也內附 Personal Web Server（PWS），安裝後也能架個小站跑 ASP 過過乾癮 。

簡單易用、又人人都能寫 ASP 來執行看看，當年可真是一大熱門話題啊～！[1]

跟 PHP 分一杯羹

不過，真要架站的話，應該沒有人會為了 ASP 買有 IIS 的 Windows 2000。而 PWS 有連線限制，架個區網的網頁伺服器還可以，不適合用在網際網路全球資訊網這大場面。因此真正有興趣的人，幾乎都會轉向語法已經成熟、而且免費就能下載安裝使用的 PHP。

其實 PHP 比 ASP 更早發展，但它還叫 Personal Home Page Tools 時，比較像是應付架設個人網站所需時，不斷追加東西進去的函式庫，難免有些雜亂無章，還是語意架構得清晰易懂的 ASP 更為簡單易用[2]。另外，在 Windows 安裝 IIS 或 WPS 都很容易、一個安裝檔走完就能搞定，PHP 的話可能有設定複雜、裝不起來的問題。所以 ASP 推出之際，還是比較受到青睞。

不過，改名為 Hypertext Preprocessor 準備推 PHP3 時，PHP 整個語法的框架已經很熟成。雖然還是沒有 IIS 或 PWS 好安裝，但不想花錢的話，再難也願意去克服。於是，越來越多人把對 ASP 的青睞轉移到其它選項：尤其是 PHP 時，Java 陣營這邊也推出了名為 JSP 的 JavaServer Pages，希望跟著搶一杯羹～

JSP 夢饜

JSP 在當時是非常強而有力的選擇方案！它跟 ASP 一樣容易上手，卻可以透過 Java 引以為豪的 JDK 來拓展設計。很快 JSP 系統實作得無比強大，除了 ASP 的功能，還包山包海的把更多東西給囊括進來，簡直可以說是無敵之姿、橫空出世來形容～

結果，JSP 反而不像 ASP 是簡單就能掌握的東西！

如果你沒徹底搞懂，以為方便，就引進那些包山包海的機制，有可能引來漏洞，而產生設計上的瑕疵。當網站正式商轉，有可能發生無法彌補的營運後果。

這時經過評估，你就會覺得台灣企業繼續用容易搞定整個系統都在做什麼的 PHP，才是正確的！並不是我們跟不上時代、不想跟美國各大企業改用 JSP，而是因為不曉得整個系統背後有沒有在做什麼的 JSP 其實是種風險。

或許現在寫 JSP 已經安全很多，但在 2000 年代初期，很多漏洞的原因，是 JSP 自己功能太多卻未妥善規劃安全問題。每次為了防範漏洞，寫出來的程式根本就稱不上媲美 ASP 的 JSP，而是很像 C++ 的 Java，有礙觀瞻。更別說有些漏洞的處理辦法，複雜到只有專業人士才懂，當時為了跟流行改用 JSP 的企業，不少造成了夢靨，回頭改回 PHP。

簡單易用的初衷

所以要寫 JSP，我的建議是：「請寫得像 ASP。」因為這是 JSP 的初衷。

套用各式各樣的機制，展現自己對 JSP 的博學多聞，確實讓人很有成就感！但我覺得願意寫出像 ASP 或 PHP 簡單易用的 JSP 網頁程式，倒也蠻有文藝氣質的，有點像展現網頁內容的創作家，而不是炫耀程式語法的工程師。

[1] 雖然說是熱門話題，不過應該說是少數人之間討論得很熱烈的話題；正常人不會有興趣關注伺服端網頁程式。
[2] 這裡指的語意架構是 Framework。ASP 設計之初，就已經很成熟地根據 HTTP Web Server 所需，去蕪存菁定義了適合架設網站的 Framework。

湯姆貓伺服器!? 啥米挖糕？

Tomcat 是用以支援 JSP 語言的 HTTP 伺服器系統。JSP 是以 Java 程式語言為基礎所發展出來的動態網頁程式語言，相當於 Java 語系的 ASP 和 PHP。此外開發人員將 Tomcat 用以直接執行 Servlet 的話，更能讓你對伺服端程式的開發展現更強的發展性；Servlet 是以 Java 程式語言為基礎的伺服端程式元件，相當於 Java 語系的 CGI。

前提

1. 本文章內容僅適用於 Windows 作業系統，使用 Linux 需要對許多名詞作調整。
2. 使用前您的電腦必須先安裝 Java 開發系統，並設定好相關的系統變數以便 Tomcat 能搜尋到 Java 程式引擎。

下載

請連線到 http://tomcat.apache.org 網址，在開啟的網頁中可以看到 Download 項目底下列出各種版本的 Tomcat 軟體名稱，點下超鏈結文字即可進入下載網頁。

在下載網頁中，Mirrors 標題可用來挑選下載檔案的伺服器主機，而 Binary Distributions 標題底下列出的則是已經編譯過可以被執行的檔案，請下載 .zip 類型的檔案。除了 ZIP 類型，另外一種類型是 Windows Service Installer，適用於 Windows 作業系統，提供自動化的安裝過程；自動化往往反而無法隨心所欲，因此本範例使用 ZIP 類型的檔案。

通常檔案會分為 Core 與 Deployer 兩種，由於我們下載 Tomcat 是要做為執行 JSP 網頁的 HTTP 伺服器系統，需要獨立執行，因此請選擇 Core。 ( Deployer 是將 Tomcat 做為子系統部署於其他系統用的，需要與其他接口搭配以提供 JSP 容器的功能。)

安裝與設定

將下載回來的 ZIP 檔案解壓縮，然後將解出來的資料夾重新命名為合適的檔名，並且移動到合適的位置；本文的示範則是命名為 Tomcat，然後放到 C:\Program Files\Apache Group 底下。

進入設定環境變數的視窗畫面，加入變數名稱：CATALINA_HOME，變數值設為 C:\Program Files\Apache Group\Tomcat。接著，在 path 變數名稱的值追加 %CATALINA_HOME%\bin 這筆資料。如果 path 變數已經有其它變數值，記得 %CATALINA_HOME%\bin 前面要加; 符號做為與其它變數值得區隔。

接著要設定管理者的帳戶名稱與密碼，請進入 Tomcat 所在位置底下的 conf 資料夾，編輯裡面的 tomcat-users.xml 檔案，然後加入如下敘述：

<user name="帳號" password="密碼" roles="standard, manager" />

再來設定虛擬路徑，也就是設定一個名稱讓它指向電腦硬碟中用來存放網頁的資料夾，以後便能夠在網址中用這個虛擬名稱轉而執行其它路徑裡面的網頁與檔案。請進入 Tomcat 所在位置底下的 conf 資料夾，編輯裡面的 server.xml 檔案，然後找到 元素，再繼續找裡面的 元素，最後加入如下敘述：

<Context path="/虛擬路徑名稱" docBase="硬碟路徑" />

例如當我這樣設定：

<Context path="/hello" docBase="D:\My Website" />

往後若輸入網址 http://127.0.0.1:8080/hello，Tomcat 伺服器系統便會從 D 槽的 My Website 資料夾中讀取網頁與檔案等相關資料。

如果不想設定虛擬路徑，可將網頁與檔案放置於預設的伺服器目錄 webapps 中存放，這樣便能從伺服端載入網頁。

啟動 Tomcat 伺服器系統

開啟命令提示字元，然後輸入：

catalina srart

即可啟動 Tomcat 伺服器系統。

測試 Tomcat 伺服器系統

開啟瀏覽器，在網址列輸入本機專用網址與埠號，測試是否能順利顯示 Tomcat 網頁。例如：

http://127.0.0.1:8080

關閉 Tomcat 伺服器系統

開啟命令提示字元，然後輸入：

catalina stop

即可關閉 Tomcat 伺服器系統。

其它補充

Tomcat 僅適合做為執行 JSP 的程式媒介，要做為 HTTP 伺服器則不夠堅實，因此當你準備正式地將開發於 Tomcat 底下的 JSP 系統放到網際網路運行時，建議將 Tomcat 整合到市佔率最高的 HTTP 伺服器系統 Apache 底下來運行。

學會 JSP 的話，將成為你手中的一大絕活！因為以網頁做為執行你程式的操作介面，這遠比 Java 應用程式更容易部署給用戶使用，用戶端只需安裝 Firefox 或任何主流網頁瀏覽器，就能執行你開發出來的系統，不需再為應用程式的相容性問題煩惱。

在 Linux 用 Tomcat 架 HTTP Server

請先下載 Java 與設定好 JAVA_HOME 系統變數。

Step 1: 下載與設置

在 Linux 下載 zip 檔或 tar.gz 檔的 Tomcat，直接解壓後就能使用。

Step 2: 啟動 Tomcat

進入終端機模式，切換到 Tomcat 所在路徑的 bin 目錄，輸入：

sh catalina.sh start

建議用 sudo 執行，才能使用 80 埠號。

Step 3: 查詢伺服器的網址

在終端機模式輸入 ifconfig，查看 inet addr 的值是多少，假設是 192.168.1.2，這就是你 Tomcat HTTP Server 的網址。

Step 4: 連線到 Linux 網頁伺服器

在其它電腦開啟網頁瀏覽器，於網址列輸入 192.168.1.2:8080，就能進入 Linux 用 Tomcat 架的網頁伺服器。

補充！要查看某些埠號是否被佔用的話，請輸入 netstat -an 來查詢。
