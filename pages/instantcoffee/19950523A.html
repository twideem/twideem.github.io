Java 開發套件重要紀年

1996 年 1 月 23 日：發佈 JDK 1.0，代號 Oak (橡樹)。

雖然 Sun Microsystems 在 1995 年 5 月 23 日正式發表 Java 程式語言這項技術，不過這天的主角其實是能執行 Java Applet 的 HotJava 瀏覽器，主打的是這款瀏覽器將帶來怎樣的效益，而不是 Java 有多好。

1996 年 1 月 23 日，Sun Microsystems 發表 JDK 1.0 (Java Development Kit)，這天才是真正將 Java 這項技術產品化[1]，讓這項技術普及化給一般人使用的重要日子。

JDK 的問世，讓 Java 程式語言正式成為開發工具。或許很少人料想得到，往後十數年期間，Java 成為最熱門的程式語言。

1997 年 2 月 19 日：發佈 JDK 1.1。

主要引進 AWT 套件的事件模型機制，並且提出 JavaBeans 規格的做法，對視覺化開發帶來重大的演進，一舉使 Java 變得適合開發 GUI 應用程式。

此外還增加 inner classes 的語法。

這版嘗試引進 JIT 技術[2]，能將常用的編碼暫存起來，以減少重新編譯來提高時效的方式，加強 Java 程式的執行效能。

1998 年 12 月 8 日：發佈 J2SE 1.2，代號 Playground (遊樂園)。

整個 Java 程式設計的架構重新規劃，讓 Java 的開發層級從語言 (Language) 的層次提升到平台 (Platform) 的層次，有助於開發人員在 Java 設計能有更標準的遵循原則，達到物件導向程式設計追求的一致性。

這次改變是重大的里程碑，因此取名為 Java 2，並且連同開發套件也改名為 J2SDK (Java 2 Platform Software Developing Kit)。

另外，這天也同時發佈 J2EE (Java 2 Platform Enterprise Edition)，針對企業級應用所發展出來的版本。它提供分散式架構 (distributed multi-tiered) 的規格與標準讓企業遵循，得以打造出永續維護效益的系統。至於標準用途的版本，則稱為 J2SE (Java 2 Platform Standard Edition)。

2000 年 5 月 8 日：發佈 J2SE 1.3，代號 Kestrel (隼)。

這版的重點其實是 APIs 的完整度終於令人滿意，此外引進 HotSpot 技術，能直接產生執行效率更高的程式碼，大幅提升 Java 執行效率。

2002 年 2 月 26 日：發佈 J2SE 1.4，代號 Merlin (大法師)。

這次版本的最大特色，就是以集結社群參與議題的方式來決定新版本的標準功能，好讓 JDK 標準化的腳步能按照大眾的期盼來進展，使用者最迫切需要的功能得以最先實現。例如 New I/O、Regular Expression、JAXP 這些最需要列為標準化的套件。

語法上新增 Assertion 的機制，Java Web Start 也正式囊括進來。

2004 年 9 月 30 日：發佈 J2SE 5.0，代號 Tiger (老虎)。

大幅強化 Java 語法功能以及編譯機制，包括：Generics、Metadata、Autoboxing/unboxing、Enumerations、Varargs、Enhanced for each loop、Static import。

這次又是重大的里程碑，所以這版名稱取為 J2SE 5.0。[3]

很多文章在介紹 J2SE 5.0 的新功能時容易犯一個語病：「Metadata 或者稱為 Annotations 是一種……」這讓人有 Metadata 等於 Annotations 的錯覺。這是兩種不同的東西，Metadata 是一種資料型態，而在程式設計裡，要定義 Metadata 通常透過 Annotations 這項機制來完成。所以 Annotations 不等於 Metadata，兩者只是彼此相關所以密不可分罷了！

2006 年 12 月 11 日：發佈 Java SE 6，代號 Mustang (野馬)。

強化桌面應用的功能，並加入 Annotation API、以及提供對腳本語言的整合方案。

經過幾次版本名稱感覺不對勁的折騰後，這版終於想出合適的命名：Java SE 6。

2011 年 7 月 28 日：發布 Java SE 7，代號 Dolphin (海豚)。

JVM 支援動態語言，成為多語言的程式平台。

新增語法功能：Strings in switch、Automatic resource management in try-statement、Improved type inference for generic instance creation、Simplified varargs method declaration、Binary integer literals、Allowing underscores in numeric literals、Catching multiple exception types and rethrowing exceptions with improved type checking。

這是 Oracle 收購 Sun Microsystems 後介入 Java 推出的版本，開發緩慢、功能延宕、產品不穩定、更新遲鈍，確實感覺 Java 到了 Oracle 手上越來越爛了！

2014 年 3 月 18 日：發布 Java SE 8

引進 Lambda 匿名函式句法，提升 Java 語言的實作能力。

此外，推出將檔案壓縮到 3MB 以下的 Small VM，但必須以解壓的方式執行程式，可能會降低 5% 的效率。

預計 2016 年發布的 Java SE 9

能在後端先將「程式平台」與「應用程式」放在一起裝卸，綑成一個較小模塊後，能更小巧地部署到前端使用者來執行 Java 程式的 Jigsaw 框架。

這項機制很有可能大幅改變 JRE 的舊有生態，因此標準化的難度很高，於是先從 Java SE 7 延遲到 Java SE 8 再延遲到 Java SE 9。因此開始有人呼籲說 Java 乾脆正式納入 OSGi 做為 Modularization 標準，不要再搞 Jigsaw 了！

OSGi 是已經發展十多年的 Java 模塊化標準，已經架構非常完善的機制，缺點是龐大所以複雜。目前 Apache 基金會的 Felix 就是符合 OSGi 標準，最常用來為 Java 搭建 Modularization 機制的後端平台。

題外話，Java SE 8 的 Lambda 與 Java SE 9 的 Modularization，原本就應該在 Java SE 7 實現，因為趕不上進度，延遲到後面幾個版本，所以代號依然是 Dolphin 而沒有代號。（未經實證）

[1] 產品不等於商品：「Java 是免費的產品。」
[2] 雖然 JIT 真正成熟而引進到版本裡面的是 JDK 1.2，不過在 JDK 1.1.6 嘗試使用時就已經炒得火熱而被廣泛使用。
[3] 有點彆扭的名稱，既然超越 Java 2，為何不是 Java SE 5，而是 Java 2 裡面的第 5 版。
