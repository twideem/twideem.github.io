Effective Java

創建和銷毀物件

01、考慮用靜態工廠方法 (static factory methods) 代替建構子。
02、遇到多個建構子參數時要考慮用 Builder 設計模式。
03、用私有建構子或者列舉類型強化 Singleton 屬性。
04、通過私有建構子強化不可實例化的能力。
05、避免建立不必要的物件。
06、消除過期的物件參照。
07、避免使用 finalizer()。

對於所有對像都通用的方法

08、覆寫 equals 時請遵守通用約定。
09、覆寫 equals 時也覆寫 hashCode。
10、總是覆寫 toString。
11、謹慎覆寫 clone。
12、考慮實現 Comparable 介面。

類別和介面

13、使類別和成員的關聯性最小化。
14、在公開類別中使用訪問方法而非公開屬性。
15、使非可變性最小化。
16、複合優先於繼承。
17、要麼為繼承而設計，並提供文檔說明，要麼就禁止繼承。
18、介面優於抽像類別。
19、介面只用於定義型別。
20、類別層次優於標籤類別 (Tagged classes)。
21、用函數物件表示策略 (Strategy)。
22、優先考慮靜態成員類別。

泛型

23、請不要在新代碼中使用原生態類型。
24、消除非受檢警告。
25、列表優先於數組。
26、優先考慮泛型。
27、優先考慮泛型方法。
28、利用有限制通配符來提升 API 的靈活性。
29、優先考慮類型安全的異構容器。

列舉和註釋 (Annotations)

30、用 enum 代替 int 常量。
31、用實體欄位代替序數。
32、用 EnumSet 代替位元欄位。
33、用 EnumMap 代替序數索引。
34、用介面模擬可伸縮的列舉。
35、註解優先於命名模式。
36、堅持使用 Override 註解。
37、用標記介面定義類型。

方法

38、檢查參數的有效性。
39、必要時進行保護性拷貝。
40、謹慎設計方法簽名。
41、慎用重載。
42、慎用可變參數 (varargs)。
43、返回空的陣列或者群集，而不是 null。
44、為所有導出的 API 元素編寫文檔註釋。

通用程序設計

45、將區域變數的作用域最小化。
46、for-each 循環優先於傳統的 for 循環。
47、了解和使用程式庫。
48、如果需要精確的答案，請避免使用 float 和 double。
49、基本資料型態優先於自動裝箱處理的類型。
50、如果其他類型更適合，則盡量避免使用字符串。
51、了解字符串連接的性能。
52、通過介面參照物件。
53、介面優先於反射機制。
54、謹慎地使用本地方法。
55、謹慎地進行優化。
56、遵守普遍接受的命名慣例。

例外

57、只針對例外的條件才使用例外。
58、對可恢復的條件使用受檢例外，對編程錯誤使用運行時例外。
59、避免不必要地使用受檢的例外。
60、盡量使用標準的例外。
61、拋出與抽象相對應的例外。
62、每個方法拋出的所有例外都要歸檔內容。
63、在細節消息中包含失敗－捕獲信息。
64、不要錯過失效時發生的警訊。
65、不要忽略例外。

平行化 (Concurrency)

66、同步訪問共享的可變數據。
67、避免過多同步。
68、executor 和 task 優先於執行緒。
69、平行工具優先於 wait 和 notify。
70、執行緒安全性的文件化。
71、慎用延遲初始化。
72、不要依賴於執行緒排程 (thread scheduler)。
73、避免整堆執行緒。

序列化

74、謹慎地實現 Serializable。
75、考慮使用自定義的序列化形式。
76、保護性地編寫 readObject 方法。
77、對於實例控制，列舉類型優先於 readResolve。
78、考慮用序列化代理代替序列化實例。
