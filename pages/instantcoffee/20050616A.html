Java 可以使用「(型別) 類別」的格式，強制切換型別：


明明編譯器知道這樣執行時會 ClassCastException，但因為使用 (String) 明確表示我要這麼做，所以允許我通過編譯。
需要注意的是，雖然這是「強制轉型」的動作沒錯，但其實目的是要告訴編譯器：「通過編譯吧！別再跟我說型別檢查錯誤，我確實要這樣幹。」它的存在並不是為了「物件 A 轉成物件 B」而來。

使用時想作「切換型別」，而不是「轉換型別」，否則你很容易誤用這個功能。

將「物件 A 轉成物件 B」應該由 class 設計 method 來完成，例如 String.valueOf() 或 Integer.parseInt() 就是很好的例子！[1]

而 (String) 或 (int) 是在編譯器所認定的物件繼承體系與你的認知不同時，用來切換物件的上下層繼承關係，嚴格來講並不是將「物件 A 轉成物件 B」的動作。

[1] 雖然這是將物件的內部結構作個轉換，感覺上轉換的不是外部型別。但外部型別應該是「切換」才對，內部結構才能「轉換」，也才是實際上的「轉型」。
