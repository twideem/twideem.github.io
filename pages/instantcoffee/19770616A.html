Java 與 JavaScript 不再免費？

Java

Java 6、Java 7、Java 8 的免費（內容已過時，沒有看的必要，想看另外按）

如果你執行 Java 時，下 -XX:+UnlockCommercialFeature 參數啟用 Oracle Java SE Advanced & Suite Products 和 Java SE Advanced Desktop 功能，或者 Oracle 認為你不是在一般用途的電腦運算（General Purpose）使用 Java SE，是要向 Oracle 支付費用的。

Oracle Java SE Advanced 系列產品功能算是個陷阱，它隨附在免費下載的 Java SE 裡面，只要用到就該付費。通常你不會下那落落長的參數啟用付費功能，但開發應用軟體的人如果下這參數給你用，Oracle 的規定是使用者付費？還是開發商付費？照常理當然是找開發商付費，是他要用到商業功能來設計軟體的。但 Oracle 並不是照常理收費的公司，而是一家認為真正在用商業功能的人是使用者，所以是找使用者付費。

而一般用途運算，大致上原則是在桌上型電腦、筆記型電腦、平板電腦、伺服器開發 Java 程式來執行是免費的，如果要將 JVM 當嵌入式系統拿到其它設備使用，才必須向 Oracle 洽談費用事宜。這不難理解，因為 Oracle 只有為個人電腦提供 Java SE，如果想在沒 Java SE 的裝置使用 Java，本來就只能找官方洽談技術支援。如果有監視器、錄放影機、收銀機、自動提款機、行動電話是跑 Java 程式的，想也知道要授權才能合法使用，Oracle 根本沒有提供這些裝置的 Java SE。[1]

舉些實例來看吧：

1、用 JDK 寫 Java Application，放上網分享給大家用 JRE 執行，屬於在 PC 開發 Java 程式來執行的用途，符合規定。
2、架設伺服器時用 JVM 當程式引擎，就像拿 Perl 直譯器當 CGI 程式引擎一樣，寫寫程式、載入 API 來用，也是符合規定的用途。
3、將 JVM 拿到非 PC 裝置當嵌入式系統，或將 JVM 改成伺服器系統讓客端電腦多工操作，則不符合 JVM 一般用途運算的定義。
總之不用太擔心，你在個人電腦桌面環境使用的 Java SE 依然免費，就跟以前 Sun Microsystems 一樣沒變，Oracle 也沒價目表，根本不想向你收費。有價目表的，就是你啟用了 Oracle Java SE Advanced 系列產品，個人桌面使用大概台幣兩千塊，整家公司使用大概一萬五。

一般用途運算外，想用 Java 技術的話，除了向官方洽談技術支援，就是改用 OpenJDK 的原始碼自行編譯，不要安裝 JRE 來執行 Java 程式。這也是從 Sun Microsystems 時代就這樣做，只是大多由廠商主動商談授權費，不主動的話官方不積極追討，除非像 Blu-ray Disc 這種有名的東西，才有積極向播放器廠商收取費用[2]。

Java 11、Java 8u211 的不再免費

Java SE 11 和 Java SE 8 Update 211 開始，全面使用 OTN 協議（Oracle Technology Network License Agreement），取代舊的 BCL 協議（Oracle Binary Code License Agreement），Oracle JDK 確定不再免費，屬於向官方取得授權然後商業使用的軟體，Oracle 要求一般免費使用者選擇 OpenJDK。這裡有個陷阱，Oracle JDK 依然是免費下載的，但只能評估和測試用，正式用在企業營運得向官方洽談授權費用。

換句話說，現在連「一般用途的電腦運算（General Purpose）」都不成立[3]，只要是企業用、有營收，就不可能免費使用 Oracle JDK！若只在自己電腦使用，Oracle 會視為評估和測試用途，不會向你收錢。但不要公開散佈或在網路運轉你用 Oracle JDK 產生的 Java 應用程式或系統，否則 Oracle 片面認定你販售或營利該 Java 程式的話，就準備催收了！因為你已經不是在評估和測試 Oracle JDK，你有在用 Oracle JDK 營利了～

寫的 Java 程式不能散佈和公開，那不就跟沒寫一樣？所以以後想寫 Java 程式，請用 OpenJDK，不要用 Oracle JDK。可以說 Oracle JDK 並不是過去的 Java SE Development Kit，而是 Oracle 的 JDK 產品，是給企業用的，而不是給大家下載使用的免費軟體資源。

也因此，你可以發現到，Java SE 11 開始，Oracle 官網不再提供 JRE 了，只有 JDK，明確告訴大家他們的 Java 產品就是授權用，不再提供 JRE 這種免費開放給大眾使用的軟體了！Oracle 會繼續為 OpenJDK 提供技術研發，且承諾研發上會讓各家的 OpenJDK 都能和 Oracle JDK 一致，等於讓 OpenJDK 填補 JRE 的缺。

還是一頭霧水的話：

Java 11 開始，Oracle 官網下載的 JDK，只有評估和測試是免費的，評估完想散佈或商轉你用 Oracle JDK 做的 Java 程式，一律得向官方洽談授權費用。

Oracle 官網不再提供免費的 Java 讓大家使用，Oracle 只提供商業用途的 JDK，至於過去免費的 Java，Oracle 轉移到 OpenJDK 了！

而且是 Oracle 自己建議非 Oracle JDK 客戶的一般使用者改用 OpenJDK！為了讓 OpenJDK 能銜接過去免費開放使用的 JRE 應用，Oracle 還釋出一些 JRE 的專利給 OpenJDK，讓一般使用者仍有免費散佈和公開 Java 程式的方案，又不至於在相容性上產生問題。

結論

其實還是跟以前一樣，Java 有免費使用和商業使用兩種！

只是 Oracle 不再提供免費使用的版本，把免費使用的版本移交到 OpenJDK 而已。

所以嚴格來說，變的只有 Oracle 的業務，Java 程式語言可以免費使用的情況並沒有變！你依然可以像過去那樣免費使用 Java，只是要改用 OpenJDK，而不是 Oracle JDK。

其實這樣的改變，反而更能保障免費使用 Java 的開發者和企業！過去 Oracle 提供付費和免費兩種 Java 時，想免費使用卻會誤入商業使用的陷阱。現在 Oracle 只提供付費的 Java，免費使用的 Java 由 OpenJDK 提供，付費版和免費版界線分明，使用者不用再擔心有哪些陷阱。

JavaScript

擁有 JavaScript 商標權的 Oracle 決定捍衛自己的權益了！任何公開營運行為，未經 Oracle 授權，都不能冠上 JavaScript 這個名稱，包括網站名稱、軟體名稱。

由於只有商標權，而不是著作權：「畢竟這是 Netscape 發明的產品，只是發明完有說好由 Sun Microsystems 註冊商標。」因此你還是可以在網路提及 JavaScript 這項程式語言、使用 JavaScript 這項技術，只有公司行號、商品名稱、活動主題…之類的企業行為，不能使用 JavaScript 當招牌攬客而已。

你可以寫 JavaScript 教學文章、也可以在文章標題使用 JavaScript，但網站名稱不能用到 JavaScript 字樣，否則 Oracle 就有權寄律師函要求你改名。你可以自由、免費的使用 JavaScript 程式語言，開發 RIA 並線上營運，但 RIA 名稱不能有 JavaScript 字樣，否則 Oracle 就有權寄律師函要求你改名。

想讓大家知道你是 JavaScript 相關行業的話，可以改用 ECMAScript 字樣，因為 Ecma International 是制定標準化的組織，除非你冒充這組織行事，否則不太可能指稱你侵犯商標權。或者改用 JS 簡寫、j 縮寫，通常一看就知道是 JavaScript。但不要用 ScriptJava，你不是第一個單字倒過來放的人，這招早被判輸好幾次，行不通的～
