function (subroutine)、method、operation、behave

function：函式。
subroutine：副程式。
method：方法；本站譯為「操作方式」或「使用方法」。
operation：操作。
behave（behaviors）：行為。

function (subroutine)

傳統程式設計中，提供 function 的模組化機制，讓程式設計師把可以運算出結果的程式碼段落獨立出來。使用上就像函數一樣，代入數字，然後依公式計算出結果。

更早的程式設計，沒有傳回值的模組化機制，單純只是用個識別名稱把某段重複執行的程式碼獨立出來呼叫使用，就叫 subroutine。

method

指 class 提供的使用方法，但更簡單來說，寫在 class 裡面的 function 或 subroutine 就叫 method。

與 operation 的差別是，我們透過 method 來驅使 class，也就是 class 是被動的。

operation

指 object 具備的操作功能，比較像是提供的使用者介面，是為了因應 UML 這樣的專案管理語言而提出來使用的名詞。

與 method 的差別是，我們透過 object 執行 operation，也就是 object 是主動的。

behave

正式術語應該用 behaviors 才對。

這是站在系統生態的角度，去了解基於怎樣性格特質的物件，有怎樣的行為。

就像在研究歷史人物成功與失敗的行為表現一樣，所以我個人習慣改用動詞的 behave，而不是名詞。

白話一點，function、method、operation 的實作情形，就叫 behaviors。
