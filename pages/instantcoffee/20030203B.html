Collection 1.1

我將 Java 1.1 時引進的資料群集，稱為 Collection 1.1，它當初有效解決了 Java 陣列大小固定、無法追加資料的窘境，為開發人員的設計提供了相當大的彈性。

這些資料群集中，最主要使用的是 Vector…光看名稱就知道是抄 C++ 語言而來，可見當初有點應急湊用的意味。後來有更好、更符合 Java 精神的選擇：List、Set、Map，所以不建議繼續使用 Collection 1.1。

雖然不應該再繼續使用，但新舊交替時，Collection 1.1 因為適合用在多執行緒的場合而延續了慧命。不只如此，舊的資料群集還修改介面，能與新群集相容，所以並不是每個人都覺得 Collection 1.1 應該徹底退場。

無論如何，本章節站在舊版的歷史點，來回味 Collection 1.1 的使用方式。不過只當作懷舊而已，千萬別再自討沒趣，使用這些群集～
