共通類別庫

JavaScript 提供幾個類別，做為我們平日開發程式時的核心組件。雖然現在來看，反而 BOM 的類別用得比較多，但這也值得我們思考：「是不是目前慣用的 JavaScript 開發手法不夠健全？是否該多用共通類別來組織程式？」[1]

Array

Array 物件是 JavaScript 提供「陣列」資料結構的解決方案。雖然採行這樣的方式，只能建立一維陣列，但還是可以對陣列產生陣列的方式，來實現多維陣列的設計。

功能

concat(元素1,元素2,…)將其它陣列或資料加入陣列。
indexOf(資料)從陣列開頭搜尋資料。
join('區隔符號')以指定符號為區隔，輸出陣列資料。參數省略的話預設為 , 逗號。
lastIndexOf(資料)從陣列結尾開始搜尋資料。
pop()移除最後一筆資料。
push(資料1,資料2,…)在陣列末端加入新的資料。
reverse()顛倒陣列資料的排列順序。
shift()移除第一筆資料。
slice(開始位置,結束位置)傳回指定擷取範圍內的資料。
sort()進行排序。
splice(位置,數量,資料)隨參數不同，進行移除、插入、取代資料不同動作。
toString()以字串輸出陣列的資料，並以 , 作區隔。
unshift(資料1,資料2,…)在陣列開頭加入新的資料。
valueOf()以原型輸出 Array 物件的資料。

狀態

constructor傳回 function Array(){ [native code] }。
length取得資料的筆數。
prototype對物件添加「功能」或「性質」。

Boolean

Boolean 物件用來直接下達 true 與 false 兩種值該如何輸出，看是要直接以文字的方式輸出其值？還是先按照資料本身的性質決定如何輸出？

然而，toString() 和 valueOf() 其實是系統在內部運作時自動調用的功能，系統會根據資料的使用場合自動執行這兩個功能，原則上使用者不需要用到它，因此平常會用到 Boolean 物件的情形並不多。

但並不是不建議使用它來保存 true 和 false，只是很少有人用罷了。

功能

toString()以字串輸出 Boolean 物件的資料。
valueOf()以原型輸出 Boolean 物件的資料。

狀態

constructor傳回 function Boolean(){ [native code] }。
prototype對物件添加「功能」或「性質」。

雖然平常很少會需要用到 Boolean 物件，但它卻是很好的 JavaScript 物件教學範本。

這通常發生某外層函式呼叫內層函式時，內層認定的 this 明明是函式物件本身，但外層認定的 this 卻是 global 作用域，結果外層帶入的 this 破壞了內層所預期的 this，導致內層函式物件無法正常透過 this 指定想要的項目。這時用 bind() 就能明確傳入一份 this 作用域是誰的物件做為內層函式給外層函式執行！

RegExp

RegExp 物件用來實現 Regular Expressions，這是一種強大的字元匹配機制。台灣通常稱為「正規表示法」，白話一點是：「適合平常拿出來用的表示式。」意思是它的功能強大，所以平常寫程式時，都會想拿出來用……但過於複雜，不想拿來用的人應該也很多。

功能

compile('字串')編譯正則表達式。
exec('字串')檢索字元串中指定的值，返回找到的值，並確定其位置。
test('字串')檢索字元串中指定的值，返回 true 或 false。

狀態

globalRegExp 物件是否啟用 g 參數。
ignoreCaseRegExp 物件是否啟用 i 參數。
lastIndex一個整數，標示開始下一次匹配的字元位置。
multilineRegExp 物件是否啟用 m 參數。
source正則表達式的源文本。

表示法符號

[]尋找方括號之間的任何字元。
()尋找括弧之間的字串。
(?:)同上，但只確認匹配的結果，不做擷取資料的動作。
^在括號中使用表示條件相反。
-在括號中使用表示從…到…。
|在括號中使用表示其它項目。

元字符

String

String 物件可以對「文字資料」進行多種操作，例如：搜尋、替換、擷取。

功能

charAt(字元位置)傳回指定位置的字元。
charCodeAt(字元位置)傳回指定位置字元的 Unicode 編碼。
concat('字串','字串',…)將多個字串結合成一串。
fromCharCode(字元編碼,…)輸出 Unicode 編碼對應的字元。
indexOf('比對字串')比對是否出現所指定的字串，有的話傳回找到的字元位置。
lastIndexOf('比對字串')類似 indexOf()，但由後往前找。
match(正規表示法)比對 RegExp 條件，有的話傳回找到的資料。
replace(正規表示法,'替代字串')比對 RegExp 條件，有的話改為替代文字並傳回資料。
search(正規表示法)比對 RegExp 條件，有的話傳回找到的位置。
slice(開始位置,結束位置)擷取範圍內的字串。
split('分割點',區段)以分割點作依據，將字串切成好幾個區段。區段可省略。
substr(開始位置,長度)從指定位置開始擷取指定長度的字串。
substring(位置,位置)擷取範圍內的字串，與 slice() 差異。
toLowerCase()將英文字母以小寫輸出。
toUpperCase()將英文字母以大寫輸出。
toString()以字串輸出 String 物件的資料。
valueOf()以原型輸出 String 物件的資料。（String 的原型就是字串）

狀態

constructor傳回 function String(){ [native code] }。
length取得文字的長度。
prototype對物件添加「功能」或「性質」。

非標準功能：以 HTML 4.01 格式輸出文字

anchor()將文字以 <A NAME></A> 格式輸出
big()將文字以 <BIG></BIF> 格式輸出
blink()將文字以 <BLNK></BLINK> 格式輸出
bold()將文字以 <B></B> 格式輸出
fixed()將文字以 <TT></TT> 格式輸出
fontcolor()將文字以 <FONT COLOR=''></FONT> 格式輸出
fontsize()將文字以 <FONT SIZE=''></FONT> 格式輸出
italics()將文字以 <I></I> 格式輸出
link()將文字以 <A HREF=''></A> 格式輸出
small()將文字以 <SMALL></SMALL> 格式輸出
strike()將文字以 <STRIKE></STRIKE> 格式輸出
sub()將文字以 <SUB></SUB> 格式輸出
sup()將文字以 <SUP></SUP> 格式輸出
