甚至還有像 Firefox 這樣，在 Windows 與 Linux 作業系統兩種版本之間，所完成 HTML5 程度不同的情況。

HTML5 功能最完整的是 Opera 瀏覽器！但自從放棄自家排版引擎，改用 Google 基於 Webkit 而來的 Blink 引擎推出 Opera 15 後，原本一直是業界最先進的瀏覽器，變成功能最陽春的瀏覽器，就不想再用了：「功能簡潔跟功能簡陋是兩碼子事啊！」

但不能否認的是，自從改用 Blink 引擎，到了 Opera 20，瀏覽效率明顯超越了 Firefox 28！如果你打開瀏覽器，倚重的是雲端服務，而不是增益套件，那比 Chrome 輕巧、卻又比 Firefox 快速的 Opera，其實是非常吸引人的！

這點來講，即使我不想再用 Opera，但我認為 Opera 決策是正確的！

至於 Chrome 則活像 Goolge 置入性行銷用的綁架軟體，越來越常強迫你使用 Google+ 和 Google Play 的功能，然後關也關不掉。所以我大概從 2012 年開始，又從 Google Chrome 跳回 Mozilla Firefox 了！（就算要測試，也會找 Chromium。）

雖然我買了一台 Windows 8 的電腦，所以發現到 IE10 的整體排版效率，其實高於只顧 JavaScript 飆速的 Chrome，但我還是繼續用速度慢但自訂化程度高的 Firefox 比較順手。而且從 IE9 開始，也不難看出 Micorsoft 假 HTML5 標準，暗地裡還是幹一堆故意與其它瀏覽器互不相容的事情，再快也不屑用它。只是從整體排版效率來優化瀏覽器的做法，是值得 Chrome 與 Firefox 省思的！因為 IE10 也跟著你競飆 JavaScript 速度後，現在只輸 0.0X 秒感覺不出來，但整體性能卻贏你超過 1 秒甚至 3 秒，所以 IE10 反而比 Chrome 23 快的感覺就很明顯。

[*] 2016 年了，Chrome、Edge、Opera、Safari 都已支援日期與時間的控制項。IE 已經被 Microsoft 淘汰，不再增加功能，所以不支援很正常。Firefox 還在爭主流瀏覽器市場，三年過去了還不支援，身為推廣者、並建議瀏覽者用 Firefox 逛本站的我深感羞愧。
