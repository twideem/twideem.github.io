解決文字過淡的問題

Windows 的細明體在某些字體大小下，文字會顯得支離破碎，看起來很淡、糊糊的，沒有細明體清晰、銳利的效果。

解決辦法是不要用細明體顯示文字，可修改瀏覽器改用別的字型顯示文字，按：

左上角選單→工具→設定→網頁

然後把預設字型全改為微軟正黑體即可。

然而，有的網站會用 CSS 指定細明體，這樣就一定會套用細明體，而不是瀏覽器預設的字型，那就沒辦法了。只能下載修改 CSS 的延伸套件，強制所有網站都用你想要的字型。

允許自動播放音樂

在網址列輸入 chrome://flags/#autoplay-policy，將 Autoplay policy 設為 No user gesture is required。

但 Chromium 76 開始取消這選項了！所以 Vivaldi 2.7 開始，只能修改 Vivaldi 的捷徑，在目標後面加上 --autoplay-policy=no-user-gesture-required。但這招能撐多久也不知道，所以說軟平台是脆弱的，功能一改再改，讓寫程式的人無所適從。

取消下載後的各種安全性確認

在網址列輸入 chrome://flags/#enable-safe-browsing-api-download-verdicts，將 Default 設為 Disabled。

在網址列輸入 chrome://flags/#treat-unsafe-downloads-as-active-content，將 Default 設為 Disabled。

關閉通知

在網址列輸入 chrome://settings/content/notifications，將傳送前詢問我改為已封鎖。

關閉位置

在網址列輸入 chrome://settings/content/location，將傳送前詢問我改為已封鎖。

關閉翻譯

在網址列輸入 chrome://settings/languages，將「詢問是否將網頁翻譯成你慣用的語言」關掉。

修改使用者名稱

chrome://settings/manageProfile

取消關閉時繼續執行背景應用程式

在網址列輸入 chrome://settings，展開進階的項目，將「Vivaldi 關閉時繼續執行背景應用程式」取消。

假如例如拿 Vivaldi 當 PDF 閱讀器，瀏覽器經常開開關關的話，啟用這項功能會常常發生怎麼按也打不開瀏覽器的情況，因為程式還在背景執行，無法馬上重新開啟文件。所以我傾向關掉它。

BlockTube

在程式的執行方面，要比 Video Blocker 來得出色，比較不會明顯拖累 YouTube 的運作，而且阻擋率更好。

要過濾關鍵字的話，得在 Video title 使用 Regular Expressions 語法：

/關鍵字一|關鍵字二|關鍵字三|/i

否則在 Video title 一行一行寫，必須輸入完整片名才會阻擋。

Advanced Font Settings

Vivaldi 3.7 開始，正體中文翻譯沒做好，很多都變英文，甚至調用簡體中文。

與其中英混雜，不如切換到英文介面。沒想到會影響字體，無法正常顯示字體較粗的 Source Han Sans TC，都是字體過淡，看了很傷眼。

於是只好加載個延伸套件解決問題了～

Script 選擇 Traditional Han，然後 Standard、Serif、SansSerif 選擇 Source Han Sans TC，Fixed 選擇 Source Han Sans HW TC，最後按 Apply setting 鈕，就跟正體中文介面時一樣了。

復原工具列

將工具列上的按鈕全部移除的話，工具列也會整個移除，所以無法再從工具列按滑鼠右鍵選單執行復原工具列。

這功能可以在「設定」中找到，只是名稱不一樣：

左上角選單→工具→設定→外觀→重設導航工具列

舊版下載

Vivaldi 的 bug 蠻多的，而且大多不是 Vivaldi 自己的 bug，而是 Chromium 產生的。加上 Chromium 不是自家引擎、也沒有主導權，有 bug 時幾乎沒法修，所以經常需要退回舊版：

http://vivaldi.com/download/archive/
http://vivaldi.club/p/2/

像 2.6 打中文吃字的問題，到 2.8 了不但沒真正解決，反而把字吃掉的問題，進一步提升為整個版位錯亂，連字打在哪都不知道，只能退回 2.5 了！這問題在中英文夾雜半形空白時經常發生，也就是中文打字時最好不要用空白。還有 Vivaldi 2.10 因為更新到 Chromium 79，所以出現單一容器（例如 SECTION）夾雜太多元素時（例如 B 標籤），網頁載入會很慢很慢的問題，Microsoft Edge 79 也是同樣情形。這些 bug 都有夠低級的～

從 2.6 以上退回 2.5，會發生無法自動登入帳密的問題，把 [User]\AppData\Local\Vivaldi\User Data\Default\ 的 Cookies 和 Cookies-journal 刪除即可。

從 2.10 退回 2.9 可能要清除瀏覽資料的快取，否則 Google 旗下的網站像是 YouTube 在用新分頁開啟鏈結時經常一片空白。

[*] 我改用同樣基於 Chromium 但啟動速度很快的 Microsoft Edge 了！本想繼續用內建滑鼠手勢的 Vivaldi，但開啟超鏈結的功能經常沒反應，浪費我操作的時間。沒反應的狀況是，滑鼠游標雖然移至超鏈結文字上，但並不處於超鏈結狀態，特徵是狀態列文字不會顯示網址，這時滑鼠手勢開啟超鏈結的功能不會動作。並非單純剛載入網頁時才會發生，有時游標移到第一個超鏈結是 OK 的，接著移到第二個超鏈結就會發現並未浮現網址的狀態列文字。
