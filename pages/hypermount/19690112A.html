資料

程式設計的基本做法，就是設立一堆可用來保存運算結果的資料，然後加以存取。

資料

在 JavaScript 程式設計，有以下幾種資料。

數值

可以使用 0 到 9 來組成數值，做為程式設計的資料。但是要注意是否超過了 JavaScript 所能處理的範圍，畢竟電腦是有極限的，太大的天文數字其實無法計算～

特殊情況下，可能會用 NaN 表示「非數值」。

文字

如果資料是文字的話，會用兩個 " 符號（雙引號）將文字包起來，例如：

"Hello, world!"

有趣的是，如果我的文字就是 " 符號怎麼辦？這時要用所謂的 escape sequence（跳脫字元）：

\"

例如 <a href="google.com">Google</a> 寫成：

"<a href=\"google.com\">Google</a>"

雖然很醜，但這是無可奈何的事，進入程式設計領域，你必須訓練自己看習慣～

既然可用 \ 倒斜線加上 " 表示原本無法處理的符號，那為何不加入更多特殊字元到倒斜線來用？這想法太棒了！於是我們有更多 escape sequence 可用：

\'單引號。
\\倒斜線，夠冏吧，這算不算自婊？
\b表示 Backspace 字元。
\n表示換行字元。
\r表示 Enter 字元。
\t表示 Tab 字元。

雖然還有更多，但常常特殊到顯示出來 Windows 也沒反應，有些字元是別的作業系統才有在用的，所以就不全列出來了～

單引號的部分值得進一步說明。傳統程式設計的文字資料，通常分為「單一字元」以及把多個單一字元串在一起的「字串」，其中單引號表示「字元資料」，然後雙引號表示「字串資料」，例如：

'H' 'e' 'l' 'l' 'o'
"Hello"
但 JavaScript 會自動轉型，不需要嚴苛的資料型態觀念，因此可用單引號取代雙引號表示字串，所以下面的寫法在 JavaScript 是正常的：

'Hello'

而且在一份 HTML 文件就是以雙引號表示設定值的情況下，也建議使用單引號表示文字資料，避免與 HTML 的雙引號混搭造成辨識上的困擾。

開關

有時候會使用 false 和 true 表示電腦工學的 0（關）和 1（開）。

雖然我們不見得非得用 true 和 false 表示開與關的狀態，例如我們可用字串 '開' '關' 表示，端視我們自己的行事風格。但進行「數位邏輯運算」時，JavaScript 都是以 true 和 false 做為資料。

位址

「記憶體位址」也是一種資料可以在程式設計使用！

在 JavaScript 裡，宣告一個變數或者產生一個物件時，就會配置一個記憶體空間，然後回傳該空間的編號位址，讓我們可以保存起來，做為往後存取該記憶體空間的依據。

題外話，在傳統程式設計中，往往可以自行指定究竟是哪個記憶體位址，然後直接修改記憶體裡面的資料，這種做法稱為「指標」。在過去組合語言的時代，原本就是要自己負責由哪個腳位輸出入資料，因此「指標」的存在是自然而然要有的功能。但現代的電腦都是透過「作業系統」來操作，記憶體區塊必須保護起來不允許修改，不然隨隨便便就能改壞作業系統的狀態造成當機。因此「指標」到了現代成為一種危險的功能，甚至二十世紀 90 年代起的高階程式語言紛紛禁用「指標」。

因此，JavaScript 只能間接由系統傳回記憶體位址，讓我們存取資料，無法直接指定說要跳轉到哪個位址來存取～

變數

如果建立的資料，可以在程式執行期間修改，就叫做「變數」。

瀏覽器關閉後，保存於變數的資料會消失，除非你另外保存起來。甚至在程式執行過程中，離開了區域變數的作用範疇（scope），原本保存在裡面的資料也會消失，這類議題稱為變數的「生命週期（lifetime）」。

換句話說，undefined 和 null 意思也不一樣，undefined 表示還沒有資料，null 表示清空資料。undefined 本身就是名叫 undefined 的資料型態，unll 的資料型態是 object。

常數

設立的資料不允許在程式執行期間被更動，就叫做「常數」，只能讀取它的資料。[1]

宣告常數

const 變數名稱 = 初始值;

只能在建立常數的同時，透過初始值設定它的資料，此後便無法再存入資料。

除了無法存入資料外，使用上跟「變數」是一樣的。

型別

資料是有分「種類」的，例如「數值」和「文字」就是兩種不同的資料。

將不同種類的資料放在一起「運算」，雖然不見得會導致程式故障，但產生出來的結果往往不是我們想要的。例如拿數值 123 用 + 符號來加上文字 abc，就是不對盤的事情，避免這種程式是良好的習慣[2]。

因此，認識資料有哪些種類、判斷資料的種類、甚至轉換資料的種類，是 JavaScript 程式設計非常重要的議題。

資料的種類

boolean
number
object
string
undefined

boolean 這種資料類似開關，只能存入 true 和 false 兩種值，想像成 on 和 off 就容易理解了。它通常用來處理「邏輯運算」的資料，當運算結果成立，就傳回 true，不成立就傳回 false。

number 的資料可以做為數值來使用。

object 表示這資料是網頁元素，這種資料你必須研究它有哪些操作的方式，才能正確使用。

string 的資料可以做為文字來使用。

undefined 原本表示這筆資料還沒定義，但到了 ECMAScript 6 不允許取出未宣告的變數，因此變成還沒賦值的意思。

一般程式設計，資料的種類必須被固定住，例如某項資料的種類是數值的話，就只能存放述職的資料，如果存放文字資料會發生錯誤。但 JavaScript 的資料是可以任意放入的，這點必須格外注意，你必須花時間思索這種做法會帶來哪些影響。

或者換個角度來講，JavaScript 其實沒有資料型態的觀念，資料的種類只根據變數的值來分別。
