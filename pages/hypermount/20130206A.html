Colorful Candy Theme

修改 Alex Gorbatchev 的 SyntaxHighlighter 程式碼，所重新發佈的主題，以沿用 GPL 範圍。

修改部分

基本上朝向減少操作步驟的方向來修改，並以離線瀏覽的需求來包裝。

１、sh.js

把 SyntaxHighlighter.all() 寫在 shCore.js 裡面，並改檔名為 sh.js，這樣只需載入 JavaScript 檔案即可使用，不需每張網頁都寫呼叫程式。

並新增 sh.lang() 來追加語言，以及 sh.theme() 來切換主題。

２、theme/candy.css

新增 Candy 主題，並做為預設。

３、lang/*.js

將 shBrushCss.js、shBrushJScript.js、shBrushXml.js 三個檔案，集中為一個 ria.js，並新增 Plain Text 用的 lang。

附帶 c++.js、java.js、python.js 三個程式語言，請使用 sh.lang('lang/c++.js')、sh.lang('lang/java.js')、sh.lang('lang/python.js') 載入使用。

追加識別文字，例如將 Java 的 java.lang.* 類別，以及 JavaScript 的 BOM 物件，以 constants 樣式的高亮度文字來顯示。因此以 Colorful Candy 主題為例，你會看到 Java 的 String 以及 JavaScript 的 document 以斜體表示。

新增 class='brush:xxx' 叫用的名字為副檔名，例如 brush:javascript 可以改用 brush:js。

如何使用

基本用法

在網頁的 HEAD 標籤，依你檔案路徑位置，載入 sh.js 即可，例如：

<scirpt src='sh.js'></scirpt>

然後用 pre 標籤來書寫程式碼，並指定 class 為對應的語言，就會以高亮度文字來顯示，格式如下：

<pre class='brush:js'>
　…
</pre>

追加語言

在 script 標籤內，以如下語法追加原 SyntaxHighlighter 官方提供的語言。

sh.lang('http://alexgorbatchev.com/pub/sh/current/scripts/shBrushCpp.js');

切換到官方的主題

在 script 標籤內，以如下語法追加原 SyntaxHighlighter 官方提供的主題。

sh.theme('http://alexgorbatchev.com/pub/sh/current/themes/shCoreEmacs.js');

官方提供的主題有：

shCoreDjango.css
shCoreEclipse.css
shCoreEmacs.css
shCoreFadeToGrey.css
shCoreMDUltra.css
shCoreMidnight.css
shCoreRDark.css
要復原 Colorful Candy 主題的話，依你檔案路徑指向 candy.css 即可。

其它

除了以上功能，其它設定方式與 SyntaxHighlighter 相容，請參考官方的《Configuration》文件，或者本站的《如何使用 SyntaxHighlighter 顯示程式範例》介紹。
